# 提示包编译模板 — 核心知识

## 转换公式

```
分镜面板
  → 步骤 1：提取（时间、动作、情绪、角色、环境）
  → 步骤 2：转换为电影语法（景别、镜头、角度、灯光）
  → 步骤 3：添加运动层（身体、面部、环境、摄影机）
  → 步骤 4：添加连续性锁定（同一角色、服装、场景）
  → 步骤 5：压缩为 AI 图像提示词语言
  = 分镜图像提示词
```

## 标准图像提示词结构

### 1. 上下文锁定
```
延续自分镜 Part X。
保持同一角色身份、服装、环境和灯光连续性。
```

### 2. 场景定义
时长、地点、时间、氛围

### 3. 角色运动
身份描述、身体运动、面部表情变化、眼神方向

### 4. 摄影机行为
景别、角度、运动、镜头行为、取景、情绪意图

### 5. 灯光设计
色调、强度、方向、情绪映射

### 6. 环境运动
天气、粒子、光线行为、背景运动、空间深度

### 7. 声音方向（可选）
环境音、情绪音效提示、静默使用

### 8. 负面约束
```
无文字、无字幕、无水印、无 Logo。
无面部变形、无身份变更、无面部漂移。
无服装变更、无发型变更。
无新增角色出现。
无场景重置、无环境瞬移。
无不自然运动、无 CGI 伪影。
```

## 镜头时长规则

| 总时长 | 镜头数 | 每镜头时长 |
|---------------|------------|-------------------|
| 15s | 5–7 个镜头 | 2–3s |
| 30s | 8–12 个镜头 | 2–4s |
| 60s | 12–20 个镜头 | 3–5s |
| 3 min | 25–40 个镜头 | 4–6s |

规则：
- 单个镜头不超过 5 秒（情绪慢镜除外）
- 每个镜头 = 一个动作单元
- 一个镜头内不包含多个事件

## 六大核心规则

1. **动作优先**：描述物理运动，而非情绪标签
2. **摄影机明确**：每条提示词包含景别 + 运动 + 角度
3. **时间压缩**：一个镜头 = 一个动作单元（2-3s）
4. **角色锁定**：每条提示词中声明"同一角色身份"
5. **环境生动**：每个镜头包含雨/风/光/粒子
6. **镜头有因**：每次运动都有情绪动机

## 表演翻译

❌ "她很悲伤"
✔ "她低下头，肩膀下垂，呼吸放缓，眼睛回避对视"

❌ "他很紧张"
✔ "他的手指敲击桌面，反复瞥向门口，下颌紧绷"

AI 图像模型需要物理动作，而非情绪标签。

## 多 Part 生成规则

超过 15 秒的视频需拆分为多个 Part：

### Part 1
- 前驱视频参考：无
- 建立世界 + 角色 + 视觉语言

### Part N（N > 1）
- 前驱视频参考：Part N-1 的输出
- 保持：角色、服装、环境、灯光逻辑、摄影机语言
- 演进：情绪状态从 Part N-1 终点继续
- 永不重启或重置——连续的电影流

## 编译工作流

```
镜头数据（分镜 + 角色 + 摄影机 + 灯光）
  → 标准化（全部视觉化，无抽象描述）
  → 冲突解决（修复矛盾之处）
  → 提示词结构化（8 段格式）
  → 压缩（最少字数，最大可执行性）
  → 分镜图像提示词
```

- [ ] 灯光是否有意义？
- [ ] 情绪是否以物理方式表达？
- [ ] 连续性是否保持？
- [ ] 提示词是否可直接执行？
