# Director Prompt Packager — 电影级短片提示包编译器

## 概览

这是 AI Film OS 管线中的**文本级编译器**，位于 STATE 4（分镜图像生成之前）。它接收 STATE 1-3 的所有设计产出——叙事结构、情绪蓝图、摄影机语言、灯光设计、角色身份锁定——并将其编译为一份完整的**电影级短片提示包**。

这份提示包是导演愿景的总文档。任何导演或制片人阅读后都能完整理解影片的：

- 每个镜头发生什么（结构化分镜设计）
- 如何拍摄（摄影机语言：景别、运动、构图）
- 光线与色彩如何驱动情绪
- 声音如何塑造节奏
- 如何分解为 Seedance 可执行的 Part（每 Part ≤ 15s）

**此技能产出文本级设计文档——不是 Seedance 2.0 / Kling 的最终可执行视频提示词。**

提示包经用户确认后，工作流进入 STATE 5（分镜蓝图图像生成）和 STATE 6（Seedance 视频提示词编译）。

可独立用于提示包编译，或由 `director-core` 在 STATE 4 调用。

## 加载资源

此技能包含捆绑参考知识。需要时加载：

- 获取提示包模板和转换公式，阅读 `references/seedance-templates.md`

## 管线位置

```
STATE 1 → 故事与情绪
STATE 2 → 视觉设计（摄影机 + 灯光）
STATE 3 → 角色锁定
STATE 4 → [本技能] 电影级提示包编译 ← 用户确认
    ↓
[路由决策：盘点资源 → 匹配模式 → 选择方案]
    ↓                        ↓
STATE 5（条件性）         直通 STATE 6
分镜蓝图生成              （跳过 STATE 5）
```

**核心原则：提示包先于图像。** 让用户先在文本层面确认完整的导演方案，再投入资源生成视觉蓝图。这可以避免因方向错误导致的返工。

## 输出边界（硬约束）

> **STATE 4 仅产出文本级导演愿景文档。绝不产出视频平台提示词。**

此硬约束防止最常见的管线错误——误将文本设计文档输入 Seedance 2.0 视频平台。

**以下视频平台名称严禁出现在任何 STATE 4 输出中：**

- Seedance 2.0
- Kling（Kling 视频模式）

**以下内容绝对不得出现在 STATE 4 输出中：**

- "在 Seedance 2.0 中按镜头生成视频"
- "直接用于 Seedance 生成"
- "Seedance 2.0 提示词"
- "按 Shot 顺序在 Seedance 生成"
- 任何暗示此输出可直接用于视频平台的表述

**正确的下游措辞：**

- "确认本提示包后，进入 STATE 5 生成分镜蓝图图像"
- "分镜蓝图图像将作为 STATE 6 的参考输入"

违反此边界将导致用户将文本设计文档误输入视频平台，造成生成失败或异常输出。

## 编译原则

> 提示包不是文字游戏。它是可执行的设计指令。

编译器将 STATE 1-3 的设计产出转化为自包含的导演文档——任何人在无外部上下文的情况下都能理解完整的影片方案。

三个编译层：

1. **创意层**（故事 + 情绪）→ 场景目的 + 叙事节奏 + 情绪曲线
2. **执行层**（摄影机 + 灯光 + 角色）→ 镜头技术规格 + 视觉连续性
3. **交付层**（Part 分解 + 平台适配）→ Seedance 可执行的分解方案

## 输入要求

编译前验证所有上游产出已就绪：

- [ ] 叙事结构 + 情绪弧线（来自 STATE 1 — director-story + director-emotion）
- [ ] 摄影机语言蓝图（来自 STATE 2 — director-camera）
- [ ] 灯光设计系统 + 色彩脚本（来自 STATE 2 — director-light）
- [ ] 角色身份锁定（来自 STATE 3 — 用户参考图或 director-character 文本定义）
- [ ] 项目元数据：总时长、画幅比例、目标平台

如有任何产出缺失，停止并路由回对应的导演技能。绝不从不完整的输入编译。

## 编译规则

### 规则 1：动作必须精确

- 每个镜头必须描述一个清晰的物理动作，而非抽象的情绪状态。
- ❌ "她很悲伤" → ✔ "肩膀下沉，下颌放松，目光移开，呼吸放缓"
- 使用 `director-character` 中的情绪→动作映射。

### 规则 2：摄影机必须明确

- 每个镜头必须包含来自 `director-camera` 的景别、角度和运动。
- 摄影机决策必须回溯到情绪意图——无动机的摄影机行为不允许出现。

### 规则 3：时长必须可执行

- 1 个镜头 = 1 个动作单元 = 2-5 秒
- 单个镜头不得包含多个时间上分离的动作

### 规则 4：角色必须锁定

- 每个镜头必须引用 `director-character` 中的角色身份锁定
- 明确声明角色名称 + 外观锁定参数
- 不得重新生成或重新描述角色——引用锁定即可

### 规则 5：Part 必须感知平台

- 每个 Part ≤ 15 秒（Seedance 单次生成长度上限）
- Part 1 建立世界基线
- Part 2+ 必须声明连续性：从上一 Part 的终点延续
- 情绪必须演进，不能重置

### 规则 6：输出平台边界

- 提示包是文本设计文档，不是视频平台指令
- 绝不写"在 Seedance 中生成"或包含 `@[ref]` 参考语法
- `@[ref]` 语法属于 STATE 6 的领域

## 验证清单

交付最终提示包前验证：

- [ ] 叙事结构完整，有清晰的因果链
- [ ] 每个镜头有明确的叙事目的
- [ ] 每个镜头有具体的物理动作
- [ ] 每个镜头已定义摄影机行为（景别 + 角度 + 运动）
- [ ] 灯光方向一致且随叙事演进
- [ ] 角色身份在每个镜头中已锁定
- [ ] 声音设计方向覆盖全片
- [ ] Part 分解方案：每段 ≤ 15s
- [ ] Part 2+ 有连续性声明
- [ ] **输出边界合规：未提及 Seedance / Kling 视频平台**
- [ ] **未包含 `@[ref]` 语法（属于 STATE 6 领域）**
- [ ] 所需视觉资源清单完整

## 下游集成

此技能产出文本级提示包后，工作流继续：

1. **用户确认提示包** → 所有分镜设计、摄影机语言和声音方向已确认
2. **路由决策** → 根据资源选择方案（A: STATE 5 分镜蓝图 / B: 直通 STATE 6）
3. **STATE 5（条件性）** → 如选方案 A，使用 `storyboard-sketch` / `storyboard-prompt` / `storyboard-master` 生成分镜蓝图图像
4. **STATE 6** → 使用 `seedance-video-prompt` 将参考素材编译为 Seedance 2.0 可执行视频提示词

## 集成

当 `director-core` 调用时：

- 加载所有 STATE 1-3 上游产出（故事、情绪、摄影机、灯光、角色）
- 验证上游产出完整性
- 编译完整的电影级短片提示包
- 执行验证清单（含输出边界检查）
- 提交用户最终审核
- 确认后标记 STATE 4 完成，进入路由决策

## 保存输出

交付最终输出后，提示用户以带日期和主题的文件名保存：

- 用户确认提示包后，保存到 `outputs/YYYY-MM-DD-[主题]/State-4-prompt-package.md`
- 告知用户："✅ 已保存至 `outputs/YYYY-MM-DD-[主题]/State-4-prompt-package.md`"
- 提示用户下一步操作（路由至 STATE 5/6 或手动使用）
