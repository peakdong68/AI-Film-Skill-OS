# Director Character — 角色一致性引擎

## 概览

以足够的特异性设计角色身份，使 AI 视频生成器能够在各镜头间维持它。此技能产出角色一致性表——一份结构化的身份文档，锁定面部、体型、服装、行为和情绪表达，使角色在影片的每一帧中保持稳定。

AI 视频制作中最常见的失败是角色漂移（换脸/换衣/崩风格）。此技能是防范它的主要手段。

可独立用于角色设计，或由 `director-core` 在 STATE 3 调用。

## 加载资源

本技能附带以下参考知识文件，在相应场景下读取：

- 需要**角色视觉参数的完整清单**（面部/发型/体型/服装/道具的详细规格字段）时，读取 `references/character-identity.md`
- 需要**情绪到身体动作的精确映射**（头/眼/嘴/肩/手/体六部位的微动作单元库、6种核心情绪的全身体映射、微表情系统、Camera-Motion 同步规则）时，读取 `references/motion-translation.md`

## 核心原则

> AI 视频中的角色一致性不是提示词调优问题——而是身份规格问题。

AI 必须在每条提示词中接收到相同的身份参数。描述中的细微变化会导致级联漂移。此技能创建所有提示词引用的主身份文档。

## 输出结构

### 1. Character Identity Core（角色核心身份）

```
CHARACTER：[姓名]
- 叙事角色：[protagonist / antagonist / supporting / observer]
- 年龄范围：[如 25-30]
- 性别表达：[描述]
- 情绪身份：[核心情绪状态——冷静、焦虑、激烈、脆弱]
- 心理画像：[关键特质、动机、恐惧]
- 故事功能：[这个角色在叙事中完成什么？]
- 角色弧线：[从起始状态 A → 到结束状态 B]
```

### 2. Visual Identity System（视觉身份系统）

**面部系统：**
```
- 脸型：[oval / round / angular / heart / square]
- 肤色：[具体描述词，含底色参考]
- 眼型：[almond / round / hooded / monolid / deep-set]
- 眼睛颜色：[具体描述，不要写"美丽的眼睛"]
- 眉毛：[形状、粗细、弧度]
- 鼻子：[形状、鼻梁、鼻尖]
- 嘴唇：[形状、唇厚度]
- 下巴/下颌：[形状、轮廓分明度]
- 辨识特征：[疤痕、痣、雀斑、不对称]
- 面部结构备注：[整体印象、骨骼结构]
```

**发型系统：**
```
- 风格：[具体发型名称或描述]
- 长度：[以面部特征为参照]
- 颜色：[具体色调，不要写"棕色"]
- 质感：[straight / wavy / curly / coiled]
- 分缝方向：[left / right / center / none]
- 动态质感：[随角色运动的表现]
```

**体型系统：**
```
- 体型：[ectomorph / mesomorph / endomorph / 具体描述]
- 身高：[高 / 中等 / 矮——以环境线索为参照]
- 体格：[lean / athletic / soft / angular]
- 姿态：[upright / slouched / coiled / relaxed]
- 动作签名：[怎么走路、站立、小动作]
```

**服装系统：**
```
- 标志性服装：[完整描述主要服装]
- 色彩调色板：[2-4 种定义角色着装的颜色]
- 面料语言：[材质——皮革、棉、丝、科技面料]
- 合身度：[loose / tailored / oversized / structured]
- 配饰：[表、眼镜、首饰、包、武器]
- 层次：[怎么组合搭配]
```

**标志性道具：**
```
- 道具 1：[描述、哪只手/哪一侧、何时可见]
- 道具 2：[描述、意义]
```

### 3. Cinematic Behavior System（镜头行为系统）

**眼神逻辑：**
```
- 默认注视：[direct / averted / scanning / distant]
- 思考时：[右上方 / 左下方 等]
- 说谎时：[行为线索]
- 被吸引时：[视线模式]
- 受威胁时：[眼部运动]
```

**情绪→动作映射：**
```
悲伤 Sadness → 肩膀下沉、下颌降低、呼吸变慢、眼神回避
愤怒 Anger → 下颚紧绷、静止、眼睛眯起、呼吸受控
恐惧 Fear → 微冻结、轻微后撤步、快速眼动、浅呼吸
喜爱 Affection → 犹豫接近、偷看、呼吸变浅
震惊 Shock → 完全静止、眼睛睁大、嘴微张、不眨眼
喜悦 Joy → 放松姿态、发自内心的微笑到达眼睛、开放的身体语言
```

**动作签名：**
```
- 行走风格：[步幅、步速、手臂摆动、重心分布]
- 手势模式：[手势多 / 收敛 / 扩张 / 紧张小动作]
- 静止行为：[不主动移动时的状态]
- 标志性动作：[一个定义此角色的独特动作]
```

### 4. Continuity Lock System（连续性锁定）

**Hard Locks 硬锁（不可变）：**
- [ ] 面部身份——脸型、特征、比例
- [ ] 发型——风格、长度、颜色、质感
- [ ] 体型——体格、身高比例
- [ ] 服装——标志性服装、色彩调色板
- [ ] 年龄——镜头间无老化或年轻化

**Soft Variations 软变（允许变化）：**
- 面部光线（不同角度、强度）
- 情绪表达（在定义范围内）
- 摄影机角度（同一张脸的不同透视）
- 环境效果（风吹头发、雨打脸）

**Forbidden Changes 禁止变化：**
- 换脸 / AI 重新生成面部
- 随机服装变化
- 镜头间年龄跳变
- 风格切换（写实 → 动画）
- 发色或发型变化（除非剧情驱动且明确标注）

### 5. Character Sheet Prompt（角色设定图提示词）

生成角色参考表图像的提示词：

```
角色表布局：全身正面视图、侧面、3/4 视图、背面视图
面部细节：正面特写、中性表情
表情范围：中性、[情绪 1]、[情绪 2]、[情绪 3]
服装：[标志性服装] 完整展示
灯光：中性棚拍光、干净背景
风格：cinematic realism（电影级写实）、各视图保持一致
负面排除：no text, no watermark, no face distortion, no identity variance between views
```

### 6. Multi-Character System（多角色系统）

对于有多角色项目，添加：

```
角色关系图：
[A] ←→ [B]：[关系类型、权力动态、情绪基调]
[B] ←→ [C]：[关系类型、权力动态、情绪基调]

视觉对比设计：
- 角色 A vs B：[体型对比、色彩对比、造型语言对比]
- 共享光影规则：[他们如何与同一光源互动]
- 互动规则：[物理距离规范、眼神接触模式、触碰边界]
```

## 约束

- 每个角色在进入分镜前必须拥有完整的视觉身份系统。
- 面部参数必须足够具体，以至于两个不同的提示词写手会产生相同的描述。
- 情绪→动作映射必须覆盖角色在故事中经历的完整情绪范围。
- 硬锁必须在每条包含该角色的 Seedance 提示词中被显式引用。
- 多角色场景必须定义互动规则以防止 AI 混淆。

## 集成

当被 `director-core` 调用时：
- 角色身份核心和视觉身份系统成为所有下游提示词的唯一真相源
- 连续性锁定系统的硬锁必须通过 `director-seedance` 嵌入到每条 Seedance 提示词中
- 角色表必须在 STATE 4（分镜）之前由用户确认
