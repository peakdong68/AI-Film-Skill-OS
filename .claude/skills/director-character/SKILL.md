# Director Character — 角色一致性引擎

## 概览

以足够的精度设计角色身份，使 AI 视频生成器能够在镜头间保持一致。此技能产出**角色一致性表（Character Consistency Sheet）**——一份结构化身份文档，锁定面部、体型、服装、行为和情绪表达，确保角色在影片每一帧中保持稳定。

AI 视频制作中最常见的失败是角色漂移（换脸/换衣/崩风格）。此技能是抵御它的一线防线。

可独立用于角色设计，也可被 `director-core` 在 STATE 3 调用。

**重要：** 此技能产出**角色身份定义**（文本级设计文档）。它**不**产生可执行的生图提示词。如需将身份定义编译为平台可用的角色设定板提示词（MJ/Flux/即梦/可灵），请使用 `character-image-prompt` 技能。

## 加载资源

此技能附带参考知识文件。在以下情况下加载它们：
- 获取完整视觉身份参数清单与身份锁定规范，阅读 `references/character-identity.md`
- 获取微动作单元库、情绪到全身动作映射以及微表情系统，阅读 `references/motion-translation.md`

## 核心原则

> AI 视频中的角色一致性不是提示词调优问题——而是身份规格问题。

AI 必须在每条提示词中接收相同的身份参数。描述的微小差异会导致级联漂移。此技能创建所有提示词共同引用的主身份文档。

## 输出结构

### 1. 角色身份核心

```
角色：[名称]
- 叙事角色：[主角 / 对手 / 配角 / 观察者]
- 年龄范围：[如 25-30]
- 性别表达：[描述]
- 情绪身份：[核心情绪状态——平静、焦虑、激烈、脆弱]
- 心理画像：[关键特质、动机、恐惧]
- 故事功能：[此角色在叙事中完成什么？]
- 角色弧线：[从起点状态 A → 到终点状态 B]
```

### 2. 视觉身份系统

**面部系统：**
```
- 脸型：[椭圆 / 圆 / 棱角 / 心形 / 方形]
- 肤色：[带底调参考的具体描述词]
- 眼型：[杏眼 / 圆眼 / 内双 / 单眼皮 / 深眼窝]
- 眼珠颜色：[具体颜色，不要用"漂亮的眼睛"]
- 眉毛：[形状、浓密度、弧度]
- 鼻子：[形状、鼻梁、鼻尖]
- 嘴型：[形状、嘴唇饱满度]
- 下颌/下巴：[形状、清晰度]
- 辨识特征：[疤痕、痣、雀斑、不对称]
- 面部结构备注：[整体印象、骨骼结构]
```

**发型系统：**
```
- 发型：[具体剪裁名称或描述]
- 长度：[相对于面部特征的测量]
- 颜色：[具体色号，不要用"棕色"]
- 质地：[直发 / 波浪 / 卷发 / 盘绕]
- 分线方向：[左 / 右 / 中 / 无]
- 动态质感：[随角色移动时如何运动]
```

**体型系统：**
```
- 体型：[瘦长 / 中等 / 圆润 / 具体描述]
- 身高：[高 / 中等 / 矮——相对于环境参照物]
- 体态：[精瘦 / 运动 / 柔和 / 棱角分明]
- 姿势：[挺拔 / 驼背 / 紧绷 / 放松]
- 动作签名：[如何走路、站立、小动作]
```

**服装系统：**
```
- 标志服装：[主服装的完整描述]
- 配色方案：[定义角色衣橱的 2-4 种颜色]
- 面料语言：[材质——皮革、棉、丝绸、机能面料]
- 版型：[宽松 / 合身 / 落肩 / 结构感]
- 配饰：[手表、眼镜、首饰、包、武器]
- 叠穿：[单品如何组合]
```

**标志道具：**
```
- 道具 1：[描述、哪只手/哪一侧、何时可见]
- 道具 2：[描述、意义]
```

### 3. 镜头行为系统

**眼神逻辑：**
```
- 默认注视方向：[直视 / 回避 / 扫视 / 远方]
- 思考时：[上右 / 下左 / 等]
- 说谎时：[行为线索]
- 被吸引时：[目光模式]
- 被威胁时：[眼部运动]
```

**情绪→动作映射：**
```
悲伤 → 肩膀下沉、下颌放低、呼吸变慢、眼神回避
愤怒 → 下颌收紧、静止、眯眼、呼吸控制
恐惧 → 微冻结、后退半步、快速扫视、浅呼吸
好感 → 犹豫靠近、偷瞄、呼吸变浅
震惊 → 完全静止、眼睛睁大、嘴微张、不眨眼
喜悦 → 姿势放松、微笑抵达眼角、身体语言开放
```

**动作签名：**
```
- 行走风格：[步幅长度、速度、手臂摆动、重心分配]
- 手势模式：[手势多 / 克制 / 夸张 / 紧张小动作]
- 静止行为：[不主动移动时的状态]
- 标志动作：[定义一个独特的动作]
```

### 4. 连续性锁定系统

**硬锁（不可变）：**
- [ ] 面部身份——脸型、五官、比例
- [ ] 发型——长度、风格、颜色、质地
- [ ] 体型——体态、身高比例
- [ ] 服装——标志服装、配色方案
- [ ] 年龄——镜头间不变老不变年轻

**软变化（允许变化）：**
- 面部光照（不同角度、不同强度）
- 情绪表达（在定义范围内）
- 摄影机角度（同一面孔的不同视角）
- 环境效果（风吹头发、雨打面部）

**禁止变化：**
- 换脸 / AI 重新生成不同面孔
- 随机换装
- 镜头间年龄跳变
- 风格突变（写实 → 动漫）
- 发色或发型改变（除非由故事驱动且明确注明）

### 5. 角色设定图布局参考

此节提供**视图覆盖模板**——定义角色设定板应覆盖哪些视觉角度和状态。它**不是**可执行的生图提示词。

如需将完整角色身份定义编译为平台可用的生图提示词（MJ/Flux/即梦/可灵），请路由到 `character-image-prompt` 技能，该技能遵循多视角角色设计板规范。

视图覆盖要求：
```
角色设定板布局：全身正面视图、侧面视图、3/4 视图、背面视图
面部特写：正面特写、中性表情
表情范围：中性、[情绪 1]、[情绪 2]、[情绪 3]
服装：[标志服装] 完整展示
灯光：中性影棚光、干净背景
风格：电影级写实、所有视图保持一致
负面约束：无文字、无水印、无面部变形、视图间无身份差异
```

### 6. 多角色系统

对于多角色项目，添加：

```
角色关系图：
[A] ←→ [B]：[关系类型、权力动态、情绪基调]
[B] ←→ [C]：[关系类型、权力动态、情绪基调]

视觉对比设计：
- 角色 A vs B：[体型对比、色彩对比、形状语言对比]
- 共享灯光规则：[他们如何与同一光源互动]
- 互动规则：[物理距离规范、眼神接触模式、触碰边界]
```

## 约束

- 每个角色在进入分镜前必须拥有完整的视觉身份系统。
- 面部参数必须具体到不同提示词写作者能产出相同描述。
- 情绪→动作映射必须覆盖角色在故事中经历的完整情绪范围。
- 硬锁必须在每条包含该角色的 Seedance 提示词中显式引用。
- 多角色场景必须定义互动规则，防止 AI 混淆。

## 下游管线

此技能产出**角色身份定义**（文本级设计文档）。用户确认后：

1. 路由到 `character-image-prompt` → 将身份定义编译为平台可用的生图提示词
2. 用户生成角色参考图（MJ/Flux/即梦/可灵）
3. 生成的角色参考图作为 `` `角色图` `` 用于 `seedance-video-prompt`（STATE 6）

## 集成

被 `director-core` 调用时：
- 角色身份核心与视觉身份系统成为所有下游提示词的唯一真理来源
- STATE 3 完成后，路由到 `character-image-prompt` 编译生图提示词
- 连续性锁定系统的硬锁必须通过 `seedance-video-prompt` 嵌入每条 Seedance 提示词
- 角色身份定义必须经用户确认后方可进入生图提示词生成阶段
