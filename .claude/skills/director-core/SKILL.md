# Director Core — 制作状态机

## 概览

AI Film OS 管线的总控制器。确保每个 AI 电影/视频项目流经经验证的、阶段门控的制作序列——从初始创意到最终的 Seedance 就绪视频提示词。状态机防止 AI 视频制作中最常见的失败模式：在故事、视觉语言和角色身份被锁定之前就跳到提示词生成。

此技能本身不生成创意内容。它在每个阶段将任务路由到相应的子技能，并在推进前验证输出。

此技能附带参考知识文件。在以下情况下加载：

- 获取详细的状态流规则和多 Part 结构规范，阅读 `references/production-state-machine.md`

## 制作管线

```
STATE 0 → 输入采集
STATE 1 → 故事与情绪设计
STATE 2 → 视觉设计（摄影机 + 光影）
STATE 3 → 角色锁定
STATE 4 → 提示词封装（电影级短片提示包）
    ↓
[路由决策：盘点资源 → 匹配模式 → 选择方案]
    ↓                        ↓
STATE 5（条件性）         直通 STATE 6
分镜蓝图生成              （跳过 STATE 5）
    ↓                        ↓
    └────────┬───────────────┘
             ↓
STATE 6 → Seedance 视频提示词（多模式，图像引用级）
STATE 7 → 最终验证
STATE 8 → 导出就绪
```

STATE 0-4 为强制阶段，不可跳过。STATE 5 为条件性阶段——I2V（storyboard）模式时执行（主要 I2V 推荐路径）；I2V（minimal）及非 I2V 模式时跳过。STATE 6-8 为必选阶段。

**STATE 4 与 STATE 5/6 的关键区别：**

- STATE 4（`director-prompt-packager`）：编译**文本级**电影短片提示包——将故事、视觉设计、角色身份整合为结构化的分镜设计、镜头语言、声音设计和 Seedance 分解方案。输出是完整的导演愿景文档，供用户确认后作为后续阶段的设计基础。**绝不产出视频平台提示词。**
- STATE 5（条件性）：仅在需要分镜蓝图板驱动 I2V storyboard 模式时执行。将提示包编译为 AI 图像生成器可执行的分镜蓝图提示词，生成可视化分镜板图像。
- STATE 6（`seedance-video-prompt`）：编译**图像引用级**视频提示词，供 Seedance 2.0 / Kling 使用。支持 7 种模式（T2V / I2V / R2V / FLF2V / V2V / ...）。**分镜图像可选**——仅 I2V storyboard 模式需要。输出：平台可执行的视频生成提示词。

**STATE 4 → STATE 6 的职责分离是管线设计的基础。绝不可跨越。**

## 阶段锁规则

这些是硬约束。违反它们会导致最常见的 AI 视频失败（角色漂移、视觉不一致、叙事不连贯）。

| 锁                                     | 规则                                                   |
| -------------------------------------- | ------------------------------------------------------ |
| **Story Lock（故事锁）**               | 故事结构和情绪弧线必须在视觉设计前确认                 |
| **Visual Lock（视觉锁）**              | 摄影机语言和光影系统必须在提示词封装前定义             |
| **Character Lock（角色锁）**           | 角色身份定义必须在提示词封装前确认                     |
| **Package Lock（提示包锁）**           | 电影短片提示包必须经用户确认后才能进入后续阶段         |
| **Storyboard Lock（分镜锁）**          | 如经 STATE 5，分镜蓝图图像必须在 Seedance 提示词前确认 |
| **Prompt Lock（提示词锁）**            | 所有预检项必须在最终导出前通过                         |
| **Output Boundary Lock（输出边界锁）** | STATE 4 输出不得提及 Seedance/Kling 等视频平台名称     |

如果任何锁被打破，立即停止并返回到最早的未完成状态。

## 状态机

### STATE 0 — 输入采集

先判断输入质量，再决定是否调用 `director-interview`：

| 输入质量         | 判定                         | 操作                                 |
| ---------------- | ---------------------------- | ------------------------------------ |
| 创意完整         | 已含主体+动作+场景+情绪方向  | **跳过访谈**，直接收集制作参数       |
| 描述充分但缺剧情 | 有场景/风格/调性，缺叙事动作 | 调用 `director-interview` → 创意展开 |
| 创意模糊         | 仅主题词/宽泛概念            | 调用 `director-interview` → 创意访谈 |

收集制作参数（时长、风格、平台、画幅、参考素材）。如果用户说"直接生成"，用合理默认值填补并标注。

**State 0 输出**：已确认的制作简报（含创意方向）。

**产出后：** 将完整输出保存到 `outputs/YYYY-MM-DD-[主题]/State-0-brief.md`。保存后告知用户："✅ 已保存至 `outputs/YYYY-MM-DD-[主题]/State-0-brief.md`"。

**验证门控：**

- [ ] **文件已保存至 `outputs/`？**

进入 STATE 1。

### STATE 1 — 故事与情绪设计

路由到 `director-story` 和 `director-emotion`。

**必需产出物：**

- 叙事结构（三幕或五幕拆解）
- 场景列表，每个场景附带场景目的
- 情绪弧线图（平静 → 紧张 → 高潮 → 解决）
- 情绪强度时间线

**产出后：** 将完整输出保存到 `outputs/YYYY-MM-DD-[主题]/State-1-story-emotion.md`。保存后告知用户："✅ 已保存至 `outputs/YYYY-MM-DD-[主题]/State-1-story-emotion.md`"。

**验证门控：**

- [ ] 每个场景都有叙事目的
- [ ] 情绪弧线覆盖全时长
- [ ] 因果链已定义（A 导致 B 导致 C）
- [ ] 用户已确认结构
- [ ] **文件已保存至 `outputs/`？**

进入 STATE 2。

### STATE 2 — 视觉设计

路由到 `director-style`（导演风格选择）、`director-camera`（摄影机系统）和 `director-light`（灯光系统）。

**必需产出物：**

- 摄影机语言蓝图（主要镜头类型、运动词汇、角度哲学）
- 灯光设计系统（主光策略、色温弧线、反差规则）
- 色彩脚本（每幕/场景的主色、温度曲线）
- 构图规则（主要和辅助取景方式）

**产出后：** 将完整输出保存到 `outputs/YYYY-MM-DD-[主题]/State-2-visual.md`。保存后告知用户："✅ 已保存至 `outputs/YYYY-MM-DD-[主题]/State-2-visual.md`"。

**验证门控：**

- [ ] 摄影机语言与情绪基调匹配
- [ ] 灯光随叙事演进
- [ ] 色彩脚本覆盖全时长
- [ ] 用户已确认视觉语言
- [ ] **文件已保存至 `outputs/`？**

进入 STATE 3。

### STATE 3 — 角色锁定

**输入门控（先判定，再路由）：**

STATE 2 确认后，先检查 STATE 0 简报中用户是否已提供了角色/产品/道具参考图：

| 用户已有...                        | 操作                                                         |
| ---------------------------------- | ------------------------------------------------------------ |
| 角色参考图（`@[角色 ref]`）        | **跳过 `director-character`**。角色视觉身份已由参考图锁定，不需要文本级角色定义。直接将参考图注册到素材槽位，状态标记为 ✅ 已就绪 |
| 产品/道具参考图（`@[产品 ref]`）   | 同上——直接注册到素材槽位，状态标记为 ✅ 已就绪               |
| 部分有图、部分无图                 | 有图的跳过 `director-character` 直接注册槽位；无图的路由到 `director-character` 生成文本级身份定义 |
| 完全无参考图                       | 路由到 `director-character`，生成文本级角色身份定义          |

> **核心原则：参考图本身就是身份锁定。** 当用户提供了角色/产品参考图时，视觉身份已由图像锚定——不需要再用文本逐项描述面部/发型/体型/服装。`director-character` 仅在**无参考图**时调用，用于从零构建角色身份定义。

---

**路径 A：用户已有参考图（跳过 `director-character`）**

1. 将用户的角色/产品/道具参考图直接注册到素材槽位，状态 ✅
2. 输出简洁的身份摘要（从参考图中可辨识的 1-2 个关键标识），保存到 `State-3-characters.md`
3. 进入下方「角色生图询问」

**路径 B：用户无参考图（路由到 `director-character`）**

路由到 `director-character`。

**必需产出物：**

- 每个角色的角色身份定义
- 视觉锁定参数（面部、发型、体型、服装、道具）
- 行为系统（动作签名、眼神方向、情绪→动作映射）
- 多角色关系图（如适用）

**产出后：** 将完整输出保存到 `outputs/YYYY-MM-DD-[主题]/State-3-characters.md`。保存后告知用户："✅ 已保存至 `outputs/YYYY-MM-DD-[主题]/State-3-characters.md`"。

**验证门控：**

- [ ] 所有角色都有身份锁定
- [ ] 视觉参数具体到可复现
- [ ] 行为系统覆盖情绪范围
- [ ] 用户已确认所有角色身份定义
- [ ] **文件已保存至 `outputs/`？**

---

**角色身份锁定后 — 询问用户（路径 A 和 B 均执行）：**

> 角色身份已锁定。是否需要基于当前身份定义生成 Character Sheet 生图提示词？

- **需要 →** 路由到 `character-image-prompt`，将身份定义（路径 A 用参考图、路径 B 用 `director-character` 文本定义）编译为平台可用的角色设定板生图提示词（MJ/Flux/即梦/可灵）。用户随后可生成实际的 Character Sheet 图像。生成的 Character Sheet 作为 STATE 6 的 `@[图片N]` 参考输入，提升跨镜头身份一致性。
- **不需要 / 跳过 →** 跳过角色生图。路径 A 下已有参考图，身份已锁定；路径 B 下角色身份将在提示包中以文本描述形式呈现。注意：相比使用参考图，文本描述可能会降低视频输出中的身份一致性。

角色生图（如选择）与 STATE 4 提示包编译可以并行进行。

注意：路径 A 下 `character-image-prompt` 的输入是用户的参考图（模式 A：有参考图像），而非文本描述。

### 素材槽位注册表（Material Slot Registry）

STATE 3 角色锁定 + 角色生图提示词产出后，立即定义全局素材槽位。所有下游 STATE 必须引用此表。

**定义时机**：STATE 3 完成后、STATE 4 开始前。

**路径 A（用户已有参考图）的槽位示例：**

| 槽位       | 内容                    | 来源                  | 状态     |
| ---------- | ----------------------- | --------------------- | -------- |
| `@[图片1]` | [角色A] 参考图          | 用户提供              | ✅ 已就绪 |
| `@[图片2]` | [产品] 参考图           | 用户提供              | ✅ 已就绪 |
| `@[图片3]` | 分镜总览图              | State-5-storyboard.md | ⏳ 待产出 |

**路径 B（无参考图）的槽位示例：**

| 槽位       | 内容                    | 来源                  | 状态               |
| ---------- | ----------------------- | --------------------- | ------------------ |
| `@[图片1]` | [角色A] Character Sheet | character-sheets.md   | ⏳ 待用户生成      |
| `@[图片2]` | [角色B] Character Sheet | character-sheets.md   | ⏳ 待用户生成      |
| `@[图片3]` | 分镜总览图 Master Sheet | State-5-storyboard.md | ⏳ 待 STATE 5 产出 |

**规则：**

- 槽位 ID（`@[图片N]`）在全局唯一且不变，贯穿 STATE 4/5/6/8
- STATE 5 分镜总览图中，每个镜头卡片必须标注依赖的角色槽位
- STATE 6 视频提示词中，必须使用 `@[图片N]` 定义主体并引用分镜蓝图
- STATE 8 导出包中，必须包含完整的槽位→素材映射表
- 若素材尚未生成（用户未上传），使用 `@[图片N]` 占位符，附带说明"上传对应素材后生效"
- **有参考图时，槽位注册即生效。不需要用户再次上传。**

**写入检查点文件：** 槽位定义完成后更新 `STATE.md`，在 `Production Brief` 后追加 `Material Slots` 区块。

### STATE 4 — 提示词封装（电影级短片提示包）

路由到 `director-prompt-packager`。

**这是一个文本级编译器。** 它将 STATE 1-3 的所有设计产物（故事结构 + 情绪弧线 + 摄影机语言 + 灯光系统 + 角色身份）编译为一份完整的**电影级短片提示包**。这份提示包是导演愿景的总文档，包含：

- 结构化的分镜设计（逐镜头的场景、动作、情绪目的）
- 镜头语言规范（景别、角度、运动、构图）
- 灯光与色彩脚本
- 声音设计方向
- 适配 Seedance 的 Part 分解方案（每 Part ≤ 15s，多 Part 连续性绑定）

输出**不是** Seedance 视频提示词——它是下游视频生成阶段的设计基础。

**输出边界（硬约束）：**

- STATE 4 输出**严禁**提及 Seedance 2.0 / Kling 等视频平台名称
- 不得出现"在 Seedance 中生成"、"Seedance 2.0 提示词"、"按 Shot 顺序在 Seedance 生成"等表述
- 提示包内容为平台无关的导演级设计文档——不绑定特定图像或视频生成工具

**产出后：** 将完整提示包保存到 `outputs/YYYY-MM-DD-[主题]/State-4-prompt-package.md`。保存后告知用户："✅ 已保存至 `outputs/YYYY-MM-DD-[主题]/State-4-prompt-package.md`"。

**预检清单（必须全部为 YES）：**

- [ ] 故事结构已确认？
- [ ] 情绪弧线已确认？
- [ ] 视觉语言（摄影机 + 灯光）已定义？
- [ ] 角色身份已锁定并确认？
- [ ] 时长和画幅比例已锁定？
- [ ] **输出边界合规：未提及 Seedance / Kling？**
- [ ] **文件已保存至 `outputs/`？**

如有任何答案为 NO，停止并返回到缺失的阶段。

进入路由决策。

**用户下一步**：确认提示包内容。确认后不直接进入 STATE 5——先执行路由决策，确定最佳推进路径。

### STATE 4 输出后：路由决策

提示包经用户确认后，向用户呈现简洁的路由选择。不得直接进入 STATE 5 或 STATE 6。

**第一步：智能资源盘点（上下文驱动）**

路由决策前，先执行智能资源盘点——根据前期产物和上下文自动推断已有资源状态，而非展示静态清单。

**盘点逻辑：**

1. **读取 Material Slot Registry**（STATE.md 的 `Material Slots` 区块）→ 获取已注册的素材槽位及其状态
2. **检查 STATE 3 产出**：是否已调用 `character-image-prompt` 生成了 Character Sheet 提示词？→ 角色图"需用户生成"
3. **扫描故事/脚本内容**：从 STATE 1 故事结构与 STATE 4 提示包中提取——
   - 出现了多少角色？→ 每个角色对应一个角色参考图需求
   - 出现了多少具体场景/环境？→ 是否需要背景参考图
   - 是否有产品/道具？→ 是否需要产品/道具参考图
4. **检查用户原始输入**：STATE 0 时用户是否提供了任何参考素材？

**根据盘点结果生成上下文感知的资源清单，而非静态模板。** 清单格式如下，但每个项目的展示内容不同：

```
📋 资源盘点：

已有 / 就绪：
✅ 角色参考图 — [角色A] Character Sheet 提示词已产出（需用户到生图平台生成图片后变为就绪）
✅ 角色参考图 — [角色B] Character Sheet 提示词已产出（同上）
   ↳ 若用户尚未生成角色图，标注为 ⌛ 待用户生成图片

管线识别出的需要确认项：
⏳ 背景/场景参考图 — 故事涉及 [场景1]、[场景2]，是否需要提供场景参考图？
⏳ 产品/道具参考图 — 故事涉及产品 [产品名]，是否已有产品图？

用户未提及项（按需展示）：
- 首/尾帧图：暂不需要（非 FLF2V 场景）
- 视频参考片段：暂不需要
- 音频参考片段：暂不需要
```

**关键规则：**

- **"以上都没有——纯文本"仅在管线确未产出任何角色图提示词且用户未提供任何参考素材时展示**——正常情况下不应出现此选项
- **背景/场景参考图**：故事中出现具体场景名时，主动询问用户是否需要提供背景参考图。若用户跳过，在 STATE 6 以文本描述替代
- **产品/道具参考图**：故事中出现产品/道具时主动询问。若用户跳过，在 STATE 6 以文本描述替代
- **首/尾帧图、视频/音频片段**：仅在用户 STATE 0 原始输入中提及或故事结构暗示需要时才展示

**第二步：匹配模式并呈现选择**

根据盘点结果，在以下模式表中选择匹配项展示给用户（只展示相关的行，不展示全部）：

| 用户有...                | 模式                                       | 路径                                                                    |
| ------------------------ | ------------------------------------------ | ----------------------------------------------------------------------- |
| 分镜图已就绪             | I2V（storyboard）                          | STATE 4 → STATE 6                                                       |
| 参考图（角色/产品/场景） | I2V（storyboard）— 推荐；或 I2V（minimal） | STATE 4 → STATE 5 → STATE 6（storyboard）/ STATE 4 → STATE 6（minimal） |
| 仅单张参考图             | I2V（minimal）                             | STATE 4 → STATE 6                                                       |
| 首帧 + 尾帧              | FLF2V                                      | STATE 4 → STATE 6                                                       |
| 视频源片段               | V2V Edit / V2V Extend                      | STATE 4 → STATE 6                                                       |
| 仅文本（无参考资源）     | T2V                                        | STATE 4 → STATE 6                                                       |

> **对于 I2V 场景，I2V（storyboard）是主要推荐。** 它先生成分镜蓝图图像（STATE 5）以实现精确的多镜头控制。I2V（minimal）是单张参考图场景的简化备选。
>
> 你的资源：**[根据盘点生成的动态摘要]**。推荐：**[根据资源匹配的最优模式]**。你选择哪个？

**第三步：收尾确认**

用户选择方案后，将缺失资源项总结为待办清单，写回 STATE.md 的 Material Slots 区块，然后进入 STATE 5 或 STATE 6。

### STATE 5 — 分镜蓝图生成（图像级，条件性阶段）

**此阶段是所有 I2V 制作的推荐路径。** 它生成分镜蓝图图像，使 STATE 6 能够实现精确的多镜头运镜控制。仅在用户选择 I2V（minimal）单图生成时跳过，或对非 I2V 模式（T2V/R2V/FLF2V/V2V）跳过。

根据用户意图选择对应的分镜技能：

| 用户意图                                 | 路由到                 | 产出                                    |
| ---------------------------------------- | ---------------------- | --------------------------------------- |
| "生成分镜总览图/导演提案板/完整分镜蓝图" | `storyboard-master`    | 一张多镜头网格总览图（4 区或简洁板）    |
| "写一个镜头画面/这个场景怎么拍/单帧构图" | `storyboard-prompt`    | 单帧分镜提示词（8 元素框架）            |
| "做带货视频分镜/服装导演板/电商直播分镜" | `storyboard-ecommerce` | 电商板（产品区+创作者/场景区+镜头网格） |
| "规划 Seedance I2V 分镜/逐帧动作描述"    | `storyboard-sketch`    | 文本级分镜帧描述（不生图，给 I2V 用）   |

默认情况（用户未明确指定）使用 `storyboard-master` 生成总览图，因为它覆盖全片镜头序列，最适合作为 STATE 6 的分镜参考输入。如需逐镜头精细化或单帧验证，再补充调用 `storyboard-prompt`。

**这是在 STATE 4 确认后的图像生成阶段。** 将已确认的提示包中的分镜设计编译为 AI 图像生成器可执行的分镜蓝图提示词，生成可视化的分镜板图像。这些图像作为 STATE 6 的 `` `分镜图` `` 参考输入。

**核心认知：生产中只需要分镜蓝图板（一张或多张总览图），而非逐帧独立图像。** 分镜蓝图展示全片镜头序列、节奏结构和视觉语言——作为 Seedance 的叙事蓝图输入，而非逐镜头渲染素材。

**必需产出物：**

- 分镜蓝图板（1-N 张总览图，展示镜头序列 + 节奏结构）
- 逐镜头视觉描述（每个镜头的主体、构图、摄影机、灯光情绪）
- 跨帧连续性锚点

**产出后：** 将分镜蓝图保存到 `outputs/YYYY-MM-DD-[主题]/State-5-storyboard.md`。保存后告知用户："✅ 已保存至 `outputs/YYYY-MM-DD-[主题]/State-5-storyboard.md`"。

**验证门控：**

- [ ] 分镜蓝图板覆盖全片镜头
- [ ] 每帧都有叙事目的
- [ ] 角色身份在所有帧中保持一致（引用 STATE 3 锁定）
- [ ] 视觉语言与 STATE 2 定义一致
- [ ] 用户已确认分镜蓝图
- [ ] **文件已保存至 `outputs/`？**

进入 STATE 6。

### STATE 6 — Seedance 视频提示词（图像引用级）

路由到 `seedance-video-prompt`。

**这是 L5 视频生成编译器。** 根据可用参考资源选择生成模式（见下方模式选择），将参考图像/视频/音频编译为 Seedance 2.0 / Kling 平台可执行的视频提示词。

**分镜蓝图图像不是必须的——它只是 I2V storyboard 模式的一种输入。** 编译器支持七种模式：T2V / I2V minimal / I2V storyboard / R2V / FLF2V / V2V Edit / V2V Extend。

**可用输入（按模式选择所需）：**

- 分镜蓝图图像（来自 AI 图像生成器）——仅 I2V storyboard 模式需要
- 角色参考图（推荐，用于身份锁定。如用户在 STATE 3 选择跳过，则以文本描述替代）
- 产品/道具参考图（可选，用于产品锁定）
- 背景参考图（可选，用于环境锁定）
- 首帧/尾帧图像——仅 FLF2V 模式需要
- 视频片段——仅 V2V Edit/Extend 模式需要
- 音频片段（可选，用于节奏/氛围参考）

**模式选择（基于可用输入）：**

| 用户有...                      | 选择模式          | 说明                                     |
| ------------------------------ | ----------------- | ---------------------------------------- |
| 分镜蓝图图像                   | I2V（storyboard） | 通过分镜板实现完整多镜头控制             |
| 角色/产品/环境参考图           | I2V（storyboard） | 主要 I2V 推荐；需先从 STATE 5 生成分镜图 |
| 仅单张参考图（产品/角色/场景） | I2V（minimal）    | 简单的单图→视频生成；无需分镜            |
| 首帧 + 尾帧图像                | FLF2V             |                                          |
| 多种不同类型参考               | R2V               |                                          |
| 视频源片段                     | V2V Edit          |                                          |
| 视频待延续                     | V2V Extend        |                                          |
| 仅文字描述                     | T2V               |                                          |

> **对于 I2V 场景，I2V（storyboard）是主要推荐。** 它通过分镜蓝图图像（STATE 5 生成）提供最佳的多镜头控制。I2V（minimal）是单张参考图场景的简化备选。始终向用户说明两种选项及其权衡。

**产出后：** 将完整视频提示词保存到 `outputs/YYYY-MM-DD-[主题]/State-6-video-prompts.md`。保存后告知用户："✅ 已保存至 `outputs/YYYY-MM-DD-[主题]/State-6-video-prompts.md`"。

**预检清单（按模式验证）：**

- [ ] 所选模式的必需输入已就绪？
- [ ] 角色参考图可用？（或用户已明确选择跳过）
- [ ] 产品图已锁定（如适用）？
- [ ] 背景图已锁定（如适用）？
- [ ] 音乐风格和 BPM 已确定？
- [ ] **提示词字数合规（中文 ≤ 500 字）？**
- [ ] **anti-slop 检查通过？**
- [ ] **文件已保存至 `outputs/`？**

如有必需输入缺失，提示用户提供对应参考。

进入 STATE 7。

### STATE 7 — 最终验证

对所有产出物按专业制作主线进行质量检查：

**生成审核（逐 Part）：**

- [ ] 身份检查：角色/产品外观与参考图一致？
- [ ] 动作检查：动作连贯、无跳变、无形态崩坏？
- [ ] 镜头检查：运镜与指定一致，无意外跳切？
- [ ] 连续性检查：Part 间空间地理、灯光、时间流连续？
- [ ] 音频检查：口型同步、音效对位、无片尾噪音？

**全局检查：**

- **叙事检查**：完整序列是否讲述了一个连贯的故事？
- **视觉检查**：视觉语言在所有镜头中是否一致？
- **字数检查**：每条提示词中文 ≤ 500 字？
- **Anti-slop 检查**：无空洞评价词？

**产出后：** 将验证报告保存到 `outputs/YYYY-MM-DD-[主题]/State-7-verification.md`。保存后告知用户："✅ 已保存至 `outputs/YYYY-MM-DD-[主题]/State-7-verification.md`"。

**验证门控：**

- [ ] **文件已保存至 `outputs/`？**

进入 STATE 8。

### STATE 8 — 导出就绪

> **硬性规则：STATE 8 必须产出独立文件。不得将 STATE 7 验证通过视为管线终点。**

按专业交付标准打包：

> 保存到 `outputs/YYYY-MM-DD-[主题]/State-8-export.md`

**生成交付包：**

- 完整的 Seedance 2.0 提示词（按 Part 执行顺序，含每条提示词对应参考图清单）
- 图像参考角色映射（每个 `@[ref]` 对应哪张素材，角色标签映射表）
- 多 Part 连续性备注（Part 2+ 的前驱视频引用说明）
- 交付格式备注（时长、画幅比例、目标平台）

**后期交付备注（如适用）：**

- 帧率、分辨率、色彩管线（HDR/SDR）建议
- 字幕/本地化方案（字幕安全区、textless 版本导出）
- 片尾音频淡出处理提醒
- 多 Part 衔接处帧对齐建议（前段末尾删 6 帧 + 后段开头删 1 帧）
- 素材授权状态备注（角色参考图、音频片段的授权/来源说明）

**产出后：** 将完整导出包保存到 `outputs/YYYY-MM-DD-[主题]/State-8-export.md`。保存后告知用户："✅ 已保存至 `outputs/YYYY-MM-DD-[主题]/State-8-export.md`"。

**验证门控：**

- [ ] **文件已保存至 `outputs/`？**

管线完成。

**归档检查点：**

1. 将 `STATE.md` 归档到 `outputs/YYYY-MM-DD-[主题]/STATE.md`
2. 清空根目录 `STATE.md`（写入 `## Production Checkpoint\n\n无活跃项目`）
3. 告知用户："✅ 项目归档完毕。STATE.md 已归档至 outputs/ 目录，可启动新项目。"

## 会话恢复

**每次加载 director-core 时，必须首先检查进度文件是否存在。**

1. 读取 `STATE.md`
2. **文件不存在** → 从 STATE 0 开始，进入输入采集
3. **文件存在且内容为"无活跃项目"** → 从 STATE 0 开始（上一项目已归档）。可提示用户查看 `outputs/` 目录下的历史项目
4. **文件存在且有活跃项目** → 解析当前状态，同时扫描 `outputs/` 目录盘点已生成的可视化资源：

```
📋 发现制作检查点：
- 项目：[项目名]
- 当前进度：STATE N — [名称]
- 已完成：STATE 0 → STATE N-1
- 已有资源：[角色图就绪/待生成] [分镜图就绪/待生成] [视频素材/无]
- 下一步：[操作描述]

是否继续？（回复"继续"从上次断点推进，或"重新开始"清除进度）
```

**资源盘点逻辑：** 恢复会话时，检查 `outputs/` 中是否有 `character-sheets.md`（判断角色图提示词已就绪）、`State-5-storyboard.md`（判断分镜方案已产出）、用户是否在其他会话中上传了素材。资源状态写入 `STATE.md` 的 `Material Slots` 区块。

````

若用户选择"重新开始"，删除或归档旧的检查点文件，从 STATE 0 开始。

## 制作进度持久化
````
