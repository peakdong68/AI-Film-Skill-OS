# Director Core — 制作状态机

## 概览

AI Film OS 管线的总控制器。确保每个 AI 电影/视频项目流经经验证的、阶段门控的制作序列——从初始创意到最终的 Seedance 就绪视频提示词。状态机防止 AI 视频制作中最常见的失败模式：在故事、视觉语言和角色身份被锁定之前就跳到提示词生成。

此技能本身不生成创意内容。它在每个阶段将任务路由到相应的子技能，并在推进前验证输出。

## 制作管线

```
STATE 0 → 输入采集
STATE 1 → 故事与情绪设计
STATE 2 → 视觉设计（摄影机 + 光影）
STATE 3 → 角色锁定
STATE 4 → 分镜规划
STATE 5 → 提示词封装（文本级）
STATE 6 → Seedance 视频提示词（图像引用级）
STATE 7 → 最终验证
STATE 8 → 导出就绪
```

每个状态产生经过验证的产出物后，下一个状态才解锁。严禁跳过状态。

**STATE 5 与 STATE 6 的关键区别：**
- STATE 5（`director-prompt-packager`）：编译**文本级**分镜提示词包，供 AI 图像生成器（MJ/Flux/即梦）使用。输出：用于生成分镜图像的结构化文本提示词。
- STATE 6（`seedance-video-prompt`）：编译**图像引用级**视频提示词，供 Seedance 2.0 / Runway / Sora / Kling 使用。输入：已生成的分镜图像 + 角色参考图。输出：平台可执行的视频生成提示词。

## 阶段锁规则

这些是硬约束。违反它们会导致最常见的 AI 视频失败（角色漂移、视觉不一致、叙事不连贯）。

| 锁 | 规则 |
|------|------|
| **Story Lock（故事锁）** | 故事结构和情绪弧线必须在视觉设计前确认 | 
| **Visual Lock（视觉锁）** | 摄影机语言和光影系统必须在分镜前定义 |
| **Character Lock（角色锁）** | 角色身份定义必须在生图提示词生成前确认 |
| **Storyboard Lock（分镜锁）** | 所有分镜帧必须在 Seedance 提示词前确认 |
| **Prompt Lock（提示词锁）** | 所有预检项必须在最终导出前通过 |

如果任何锁被打破，立即停止并返回到最早的未完成状态。

## 状态机

### STATE 0 — 输入采集

收集制作电影所需的最低可行简报。只问开始所需的内容：

- 项目创意或剧本（至少一句话）
- 预期时长（15s / 30s / 60s / 自定义）
- 视觉风格（电影级 / 商业 / 纪录片 / 动漫 / 科幻等）
- 交付平台（Seedance / Runway / Sora / Kling）
- 画幅比例（16:9 / 9:16 / 1:1）
- 任何已有的参考图像或角色描述

如果用户提供的信息少于其中 4 项，询问缺失的部分。如果他们说"直接生成"或"just do it"，用合理的默认值填补并标注。

**State 0 输出**：已确认的制作简报。进入 STATE 1。

### STATE 1 — 故事与情绪设计

路由到 `director-story` 和 `director-emotion`。

**必需产出物：**
- 叙事结构（三幕或五幕拆解）
- 场景列表，每个场景附带场景目的
- 情绪弧线图（平静 → 紧张 → 高潮 → 解决）
- 情绪强度时间线

**验证门控：**
- [ ] 每个场景都有叙事目的
- [ ] 情绪弧线覆盖全时长
- [ ] 因果链已定义（A 导致 B 导致 C）
- [ ] 用户已确认结构

**State 1 输出**：剧本蓝图 + 情绪时间线。进入 STATE 2。

### STATE 2 — 视觉设计

路由到 `director-camera` 和 `director-light`。

**必需产出物：**
- 摄影机语言蓝图（主要镜头类型、运动词汇、角度哲学）
- 灯光设计系统（主光策略、色温弧线、反差规则）
- 色彩脚本（每幕/场景的主色、温度曲线）
- 构图规则（主要和辅助取景方式）

**验证门控：**
- [ ] 摄影机语言与情绪基调匹配
- [ ] 灯光随叙事演进
- [ ] 色彩脚本覆盖全时长
- [ ] 用户已确认视觉语言

**State 2 输出**：视觉语言蓝图。进入 STATE 3。

### STATE 3 — 角色锁定

路由到 `director-character`。

**必需产出物：**
- 每个角色的角色身份定义
- 视觉锁定参数（面部、发型、体型、服装、道具）
- 行为系统（动作签名、眼神方向、情绪→动作映射）
- 多角色关系图（如适用）

**验证门控：**
- [ ] 所有角色都有身份锁定
- [ ] 视觉参数具体到可复现
- [ ] 行为系统覆盖情绪范围
- [ ] 用户已确认所有角色身份定义

**State 3 输出**：角色身份定义（文本级设计文档）。

**STATE 3 确认后，路由到 `character-image-prompt`：**
此子技能将角色身份定义编译为平台可用的角色设定板生图提示词（MJ/Flux/即梦/可灵）。用户随后生成实际的角色参考图。这些图像作为 STATE 6（Seedance 视频提示词）的 `@[角色引用]` 输入所必需。

进入 STATE 4（分镜）——分镜与角色生图可以并行进行。

### STATE 4 — 分镜规划

路由到 `storyboard-sketch`（用于 Seedance I2V）或 `storyboard-prompt` / `storyboard-master` / `storyboard-ecommerce`（用于图像生成器分镜板）。

**必需产出物：**
- 分镜帧计划（每板 3-8 帧）
- 逐镜头拆解（摄影机、动作、情绪、目的）
- 跨所有帧的连续性锚点

**验证门控：**
- [ ] 分镜板数量与计划时长匹配
- [ ] 每帧都有叙事目的
- [ ] 角色身份在所有帧中保持一致
- [ ] 用户已确认所有分镜帧

**State 4 输出**：分镜板 + 镜头计划。进入 STATE 5。

### STATE 5 — 提示词封装（文本级）

路由到 `director-prompt-packager`。

**这是一个文本级编译器。** 它为 AI 图像生成器（Midjourney、Flux、即梦、可灵）生成结构化的提示词包。输出**不是** Seedance 视频提示词——它是生成分镜图像的输入。

**必需产出物：**
- 每个分镜帧的结构化提示词块
- 镜头间的连续性上下文锁定
- 每镜头的负面约束块
- 多 Part 连续性绑定（用于 >15 秒的视频）

**预检清单（必须全部为 YES）：**
- [ ] 所有分镜已完成？
- [ ] 用户确认了所有分镜？
- [ ] 角色定义已完成？
- [ ] 用户确认了角色？
- [ ] 视觉语言已定义？
- [ ] 时长和画幅比例已锁定？

如有任何答案为 NO，停止并返回到缺失的阶段。

**State 5 输出**：分镜提示词包（文本级，供 AI 图像生成器使用）。进入 STATE 6。

**用户下一步**：使用提示词包在 MJ/Flux/即梦 中生成分镜图像。

### STATE 6 — Seedance 视频提示词（图像引用级）

路由到 `seedance-video-prompt`。

**这是 L5 视频生成编译器。** 它接收已生成的分镜图像、角色参考图和产品/背景参考作为 `@[ref]` 输入，并将它们编译为 Seedance 2.0 / Runway / Sora / Kling 平台可执行的视频提示词。

**必需输入：**
- 已生成的分镜图像（来自 AI 图像生成器）
- 角色参考图（用于身份锁定）
- 产品参考图（可选，用于产品锁定）
- 背景参考图（可选，用于环境锁定）

**预检清单（必须全部为 YES）：**
- [ ] 分镜图像已生成？
- [ ] 角色参考图可用？
- [ ] 产品图已锁定（如适用）？
- [ ] 背景图已锁定（如适用）？
- [ ] 音乐风格和 BPM 已确定？

如有任何答案为 NO，提示用户提供缺失的参考。

**State 6 输出**：Seedance 2.0 视频提示词（平台可执行）。进入 STATE 7。

### STATE 7 — 最终验证

对所有产出物进行质量检查：

- **叙事检查**：完整序列是否讲述了一个连贯的故事？
- **视觉检查**：视觉语言在所有镜头中是否一致？
- **角色检查**：角色身份在每条提示词中是否保留？
- **连续性检查**：空间地理、灯光和时间流是否感觉连续？
- **执行检查**：Seedance 提示词是否可直接在目标平台使用？

**State 7 输出**：已验证的视频提示词。进入 STATE 8。

### STATE 8 — 导出就绪

打包最终交付物：
- 完整的 Seedance 2.0 提示词（按执行顺序）
- 图像参考角色映射（哪个 @[ref] 对应哪张图像）
- 多 Part 生成的上下文连续性备注
- 交付格式备注（时长、画幅比例、平台）

## 依赖关系图

```
director-story ────→ director-emotion
       │                    │
       └────────┬───────────┘
                ↓
         director-camera ──→ director-light
                │
                ↓
         director-character ──→ character-image-prompt [角色生图提示词 → MJ/Flux/即梦]
                │                         │
                │                         ↓ (用户生成角色参考图)
                ↓                         │
         storyboard-sketch / storyboard-prompt / storyboard-master / storyboard-ecommerce
                │                         │
                ↓                         │
         director-prompt-packager [STATE 5: 文本级提示词包 → 供图像生成器]
                │                         │
                ↓ (用户通过 MJ/Flux/即梦 生成分镜图像)
                │                         │
                └─────────┬───────────────┘
                          ↓
         seedance-video-prompt [STATE 6: 图像引用级视频提示词 → Seedance 2.0]
                          │
                          ↓
         [最终验证 → 导出]
```

## 路由指南

| 用户意图 | 首先加载 |
|---|---|
| "我有一个创意，帮我拍成电影" | 留在 director-core，从 STATE 0 开始 |
| "我已有剧本，需要视觉设计" | 从 STATE 2 进入 |
| "我已有角色身份定义，需要生图提示词" | 路由到 `character-image-prompt` |
| "我已有分镜帧（文本），需要图像提示词" | 从 STATE 5 进入 |
| "我已有分镜图像 + 角色图，需要 Seedance 2.0 提示词" | 从 STATE 6 进入 |
| "我只想要角色身份定义" | 直接路由到 director-character |
| "修复我出问题的 AI 视频，角色一直变脸" | 从 STATE 3 进入（重新锁定角色），然后 STATE 6 |
| "电商直播/带货/服装视频分镜" | 从 STATE 4 进入，路由到 storyboard-ecommerce |
