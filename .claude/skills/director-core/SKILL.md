# Director Core — 制作状态机

## 概览

AI Film OS 管线的总控制器。确保每个 AI 电影/视频项目流经经验证的、阶段门控的制作序列——从初始创意到最终的 Seedance 就绪视频提示词。状态机防止 AI 视频制作中最常见的失败模式：在故事、视觉语言和角色身份被锁定之前就跳到提示词生成。

此技能本身不生成创意内容。它在每个阶段将任务路由到相应的子技能，并在推进前验证输出。

## 加载资源

此技能附带参考知识文件。在以下情况下加载：

- 获取详细的状态流规则和多 Part 结构规范，阅读 `references/production-state-machine.md`

## 制作管线

```
STATE 0 → 输入采集
STATE 1 → 故事与情绪设计
STATE 2 → 视觉设计（摄影机 + 光影）
STATE 3 → 角色锁定
STATE 4 → 提示词封装（电影级短片提示包）
STATE 5 → 分镜蓝图生成（图像级）
STATE 6 → Seedance 视频提示词（图像引用级）
STATE 7 → 最终验证
STATE 8 → 导出就绪
```

每个状态产生经过验证的产出物后，下一个状态才解锁。严禁跳过状态。

**STATE 4 与 STATE 6 的关键区别：**

- STATE 4（`director-prompt-packager`）：编译**文本级**电影短片提示包——将故事、视觉设计、角色身份整合为结构化的分镜设计、镜头语言、声音设计和 Seedance 分解方案。输出是完整的导演愿景文档，供用户确认后作为 STATE 5 分镜蓝图的生成基础。**绝不产出视频平台提示词。**
- STATE 6（`seedance-video-prompt`）：编译**图像引用级**视频提示词，供 Seedance 2.0 / Kling 使用。输入：已生成的分镜蓝图图像 + 角色参考图。输出：平台可执行的视频生成提示词。

**STATE 4 → STATE 6 的职责分离是管线设计的基础。绝不可跨越。**

## 阶段锁规则

这些是硬约束。违反它们会导致最常见的 AI 视频失败（角色漂移、视觉不一致、叙事不连贯）。

| 锁                                     | 规则                                               |
| -------------------------------------- | -------------------------------------------------- |
| **Story Lock（故事锁）**               | 故事结构和情绪弧线必须在视觉设计前确认             |
| **Visual Lock（视觉锁）**              | 摄影机语言和光影系统必须在提示词封装前定义         |
| **Character Lock（角色锁）**           | 角色身份定义必须在提示词封装前确认                 |
| **Package Lock（提示包锁）**           | 电影短片提示包必须经用户确认后才能进入分镜蓝图生成 |
| **Storyboard Lock（分镜锁）**          | 分镜蓝图图像必须在 Seedance 提示词前确认           |
| **Prompt Lock（提示词锁）**            | 所有预检项必须在最终导出前通过                     |
| **Output Boundary Lock（输出边界锁）** | STATE 4 输出不得提及 Seedance/Kling 视频平台       |

如果任何锁被打破，立即停止并返回到最早的未完成状态。

## 状态机

### STATE 0 — 输入采集

收集制作电影所需的最低可行简报。只问开始所需的内容：

- 项目创意或剧本（至少一句话）
- 预期时长（15s / 30s / 60s / 自定义）
- 视觉风格（电影级 / 商业 / 纪录片 / 动漫 / 科幻等）
- 交付平台（Seedance / Kling）
- 画幅比例（16:9 / 9:16 / 1:1）
- 任何已有的参考图像或角色描述

如果用户提供的信息少于其中 4 项，询问缺失的部分。如果他们说"直接生成"或"just do it"，用合理的默认值填补并标注。

**State 0 输出**：已确认的制作简报。进入 STATE 1。

### STATE 1 — 故事与情绪设计

路由到 `director-story` 和 `director-emotion`。

**必需产出物：**

- 叙事结构（三幕或五幕拆解）
- 场景列表，每个场景附带场景目的
- 情绪弧线图（平静 → 紧张 → 高潮 → 解决）
- 情绪强度时间线

**验证门控：**

- [ ] 每个场景都有叙事目的
- [ ] 情绪弧线覆盖全时长
- [ ] 因果链已定义（A 导致 B 导致 C）
- [ ] 用户已确认结构

**State 1 输出**：剧本蓝图 + 情绪时间线。进入 STATE 2。

### STATE 2 — 视觉设计

路由到 `director-camera` 和 `director-light`。

**必需产出物：**

- 摄影机语言蓝图（主要镜头类型、运动词汇、角度哲学）
- 灯光设计系统（主光策略、色温弧线、反差规则）
- 色彩脚本（每幕/场景的主色、温度曲线）
- 构图规则（主要和辅助取景方式）

**验证门控：**

- [ ] 摄影机语言与情绪基调匹配
- [ ] 灯光随叙事演进
- [ ] 色彩脚本覆盖全时长
- [ ] 用户已确认视觉语言

**State 2 输出**：视觉语言蓝图。进入 STATE 3。

### STATE 3 — 角色锁定

路由到 `director-character`。

**必需产出物：**

- 每个角色的角色身份定义
- 视觉锁定参数（面部、发型、体型、服装、道具）
- 行为系统（动作签名、眼神方向、情绪→动作映射）
- 多角色关系图（如适用）

**验证门控：**

- [ ] 所有角色都有身份锁定
- [ ] 视觉参数具体到可复现
- [ ] 行为系统覆盖情绪范围
- [ ] 用户已确认所有角色身份定义

**State 3 输出**：角色身份定义（文本级设计文档）。

**STATE 3 确认后 — 询问用户：**

> 角色身份定义已锁定。是否需要生成角色参考图？

- **需要 →** 路由到 `character-image-prompt`，将身份定义编译为平台可用的角色设定板生图提示词（MJ/Flux/即梦/可灵）。用户随后生成实际的角色参考图。这些图像作为 STATE 6（Seedance 视频提示词）的 `@[角色引用]` 输入，提升跨镜头身份一致性。
- **不需要 / 跳过 →** 跳过角色生图。角色身份将在提示包中以文本描述形式呈现。注意：相比使用参考图，这可能会降低视频输出中的身份一致性。

角色生图（如选择）与 STATE 4 提示包编译可以并行进行。

### STATE 4 — 提示词封装（电影级短片提示包）

路由到 `director-prompt-packager`。

**这是一个文本级编译器。** 它将 STATE 1-3 的所有设计产物（故事结构 + 情绪弧线 + 摄影机语言 + 灯光系统 + 角色身份）编译为一份完整的**电影级短片提示包**。这份提示包是导演愿景的总文档，包含：

- 结构化的分镜设计（逐镜头的场景、动作、情绪目的）
- 镜头语言规范（景别、角度、运动、构图）
- 灯光与色彩脚本
- 声音设计方向
- 适配 Seedance 的 Part 分解方案（每 Part ≤ 15s，多 Part 连续性绑定）

输出**不是** Seedance 视频提示词——它是 STATE 5（分镜蓝图生成）和 STATE 6（Seedance 视频提示词）的设计基础。

**输出边界（硬约束）：**

- STATE 4 输出**严禁**提及 Seedance 2.0 / Kling 等视频平台名称
- 不得出现"在 Seedance 中生成"、"Seedance 2.0 提示词"、"按 Shot 顺序在 Seedance 生成"等表述
- 所有生成指令必须指向图像生成器（即梦/MJ/Flux/可灵图像模式）

**预检清单（必须全部为 YES）：**

- [ ] 故事结构已确认？
- [ ] 情绪弧线已确认？
- [ ] 视觉语言（摄影机 + 灯光）已定义？
- [ ] 角色身份已锁定并确认？
- [ ] 时长和画幅比例已锁定？
- [ ] **输出边界合规：未提及 Seedance / Kling？**

如有任何答案为 NO，停止并返回到缺失的阶段。

**State 4 输出**：电影级短片提示包（文本级导演愿景文档）。进入 STATE 5。

**用户下一步**：确认提示包内容。确认后进入 STATE 5 生成分镜蓝图图像。

### STATE 5 — 分镜蓝图生成（图像级）

路由到 `storyboard-sketch`（用于 Seedance I2V 草图）或 `storyboard-prompt` / `storyboard-master` / `storyboard-ecommerce`（用于生成完整分镜蓝图图像）。

**这是在 STATE 4 确认后的图像生成阶段。** 将已确认的提示包中的分镜设计编译为 AI 图像生成器可执行的分镜蓝图提示词，生成可视化的分镜板图像。这些图像作为 STATE 6 的 `@[分镜引用]` 输入。

**核心认知：生产中只需要分镜蓝图板（一张或多张总览图），而非逐帧独立图像。** 分镜蓝图展示全片镜头序列、节奏结构和视觉语言——作为 Seedance 的叙事蓝图输入，而非逐镜头渲染素材。

**必需产出物：**

- 分镜蓝图板（1-N 张总览图，展示镜头序列 + 节奏结构）
- 逐镜头视觉描述（每个镜头的主体、构图、摄影机、灯光情绪）
- 跨帧连续性锚点

**验证门控：**

- [ ] 分镜蓝图板覆盖全片镜头
- [ ] 每帧都有叙事目的
- [ ] 角色身份在所有帧中保持一致（引用 STATE 3 锁定）
- [ ] 视觉语言与 STATE 2 定义一致
- [ ] 用户已确认分镜蓝图

**State 5 输出**：分镜蓝图图像（供 Seedance `@[ref]` 引用）。进入 STATE 6。

### STATE 6 — Seedance 视频提示词（图像引用级）

路由到 `seedance-video-prompt`。

**这是 L5 视频生成编译器。** 它接收已生成的分镜图像、角色参考图和产品/背景参考作为 `@[ref]` 输入，并将它们编译为 Seedance 2.0 / Kling 平台可执行的视频提示词。

**平台硬约束：**

- 中文提示词 ≤ 500 字，英文提示词 ≤ 1000 词，总字符数 ≤ 2000
- 每个 @[ref] 必须分配唯一主角色（身份 / 产品 / 环境 / 动作节奏），使用角色映射声明格式
- 通过 anti-slop 检查：无空洞评价词（cinematic / epic / beautiful 等无物理指涉词汇）

**必需输入：**

- 已生成的分镜蓝图图像（来自 AI 图像生成器）
- 角色参考图（推荐，用于身份锁定。如用户在 STATE 3 选择跳过，则以文本描述替代）
- 产品参考图（可选，用于产品锁定）
- 背景参考图（可选，用于环境锁定）

**预检清单（必须全部为 YES）：**

- [ ] 分镜蓝图图像已生成？
- [ ] 角色参考图可用？（或用户已明确选择跳过）
- [ ] 产品图已锁定（如适用）？
- [ ] 背景图已锁定（如适用）？
- [ ] 音乐风格和 BPM 已确定？
- [ ] **提示词字数合规（中文 ≤ 500 字）？**
- [ ] **anti-slop 检查通过？**

如有任何答案为 NO，提示用户提供缺失的参考。

**State 6 输出**：Seedance 2.0 视频提示词（平台可执行）。进入 STATE 7。

### STATE 7 — 最终验证

对所有产出物进行质量检查：

- **叙事检查**：完整序列是否讲述了一个连贯的故事？
- **视觉检查**：视觉语言在所有镜头中是否一致？
- **角色检查**：角色身份在每条提示词中是否保留？
- **连续性检查**：空间地理、灯光和时间流是否感觉连续？
- **执行检查**：Seedance 提示词是否可直接在目标平台使用？
- **字数检查**：中文提示词是否 ≤ 500 字？

**State 7 输出**：已验证的视频提示词。进入 STATE 8。

### STATE 8 — 导出就绪

打包最终交付物：

- 完整的 Seedance 2.0 提示词（按执行顺序）
- 图像参考角色映射（哪个 @[ref] 对应哪张图像）
- 多 Part 生成的上下文连续性备注
- 交付格式备注（时长、画幅比例、平台）

| 用户意图                                                | 首先加载                                                     |
| ------------------------------------------------------- | ------------------------------------------------------------ |
| "我有一个创意，帮我拍成电影"                            | 留在 director-core，从 STATE 0 开始                          |
| "我已有剧本，需要视觉设计"                              | 从 STATE 2 进入                                              |
| "我已有角色身份定义，需要生图提示词"                    | 路由到 `character-image-prompt`                              |
| "我已有故事+视觉+角色设计，需要编译提示包"              | 从 STATE 4 进入                                              |
| "我已有提示包（文本），需要生成分镜蓝图图"              | 从 STATE 5 进入                                              |

## 会话恢复

**每次加载 director-core 时，必须首先检查进度文件是否存在。**

1. 读取 `STATE.md`
2. **文件不存在** → 从 STATE 0 开始，进入输入采集
3. **文件存在** → 解析当前状态，告知用户进度并询问：

```
📋 发现制作检查点：
- 项目：[项目名]
- 当前进度：STATE N — [名称]
- 已完成：STATE 0 → STATE N-1
- 下一步：[操作描述]

是否继续？（回复"继续"从上次断点推进，或"重新开始"清除进度）
```

若用户选择"重新开始"，删除或归档旧的检查点文件，从 STATE 0 开始。

## 制作进度持久化

制作状态通过检查点文件持久化，确保会话中断后可恢复。

### 检查点文件

- **路径**：`STATE.md`
- **写入时机**：每个 STATE 完成后立即写入；用户每次确认产出物后更新；会话结束时写入
- **何时读回**：`director-core` 每次加载时（见上方"会话恢复"）

## Production Checkpoint

- **Project**: [项目名]

- **Completed states**: STATE 0, STATE 1, ..., STATE N-1
- **Pending states**: STATE N+1, STATE N+2, ..., STATE 8
