# 制作状态机 — 核心知识

## 状态流转

```
STATE 0 → 输入采集       （收集简报，提问关键问题）
STATE 1 → 故事与情绪      （叙事结构 + 情绪弧线）
STATE 2 → 视觉设计        （摄影语法 + 灯光 + 色彩）
STATE 3 → 角色锁定        （角色设定表 + 身份锁定）
STATE 4 → 提示词封装      （电影级短片提示包编译：分镜设计 + 镜头语言 + 声音设计 + Seedance 分解方案）
STATE 5 → 分镜蓝图生成    （生成分镜故事版蓝图图像，供 @[ref] 引用）
STATE 6 → Seedance 视频提示词（图像引用级，平台可执行）
STATE 7 → 最终验证        （所有产物的质量审核）
STATE 8 → 导出就绪        （打包交付物）
```

## 绝对工作流锁

1. 故事结构和情绪弧线必须在视觉设计前确认
2. 摄影机语言和灯光系统必须在提示包编译前定义
3. 角色身份必须在提示包编译前确认
4. 提示包必须经用户确认后才能进入分镜蓝图生成
5. 分镜蓝图图像必须在 Seedance 提示词前确认
6. 不假设确认——等待用户的明确批准
7. 提示包未确认 → 分镜蓝图禁止生成
8. 分镜蓝图未确认 → Seedance 提示词禁止生成
9. 角色未确认 → Seedance 提示词禁止生成
10. STATE 4 输出严禁提及 Seedance/Runway/Sora/Kling 视频平台

## 阶段锁强制执行

| 锁 | 条件 | 违规后果 |
|----|------|---------|
| 故事锁 | 故事结构必须在视觉设计前确认 | 视觉语言不连贯 |
| 视觉锁 | 摄影机+灯光必须在提示包编译前定义 | 摄影机决策随机 |
| 角色锁 | 角色设定表必须在提示包编译前确认 | 角色漂移、换脸 |
| 提示包锁 | 提示包必须用户确认后才能进入分镜蓝图 | 分镜方向错误 |
| 分镜锁 | 分镜蓝图图像必须在 Seedance 提示词前确认 | 缺失叙事视觉参考 |
| 输出边界锁 | STATE 4 输出不得提及视频平台 | 用户误将文本提示词输入视频平台 |
| 提示词锁 | 所有预检项必须在最终导出前通过 | 连续性断裂 |

## 强制状态追踪器

每个制作会话输出必须包含：
- 当前阶段
- 已完成状态
- 待处理状态
- 活跃锁
- 下一步必要操作
- 关键锁定状态

## STATE 4 与 STATE 5 的职责分离

**STATE 4（director-prompt-packager）**：文本级编译器
- 将 STATE 1-3 的所有设计产物编译为电影级短片提示包
- 包含：结构化分镜设计 + 镜头语言 + 声音设计 + Seedance 分解方案
- 供用户确认，作为 STATE 5 分镜蓝图生成的输入基础
- **严禁在输出中提及 Seedance/Runway/Sora/Kling 视频平台**

**STATE 5（storyboard-sketch/storyboard-prompt/storyboard-master/storyboard-ecommerce）**：图像级生成
- 使用已确认的提示包生成分镜蓝图图像
- 生产中只需分镜蓝图板（总览图），而非逐帧独立图像
- 蓝图图像作为 STATE 6 的 @[ref] 输入

## 多 Part 结构规则

- 每个 Part = 一个视频片段（最长 15 秒）
- 总时长 = Part 数量 × 每 Part 秒数
- Part 2 及以上必须引用前一个输出作为连续性基线

## 预检清单（STATE 4 提示包编译前）

全部必须是"是"：
- [ ] 故事结构已确认？
- [ ] 情绪弧线已确认？
- [ ] 视觉语言（摄影机 + 灯光）已定义？
- [ ] 角色身份已锁定并确认？
- [ ] 时长和画幅比例已锁定？

## 预检清单（STATE 6 Seedance 提示词生成前）

全部必须是"是"：
- [ ] 分镜蓝图图像已生成？
- [ ] 用户已确认分镜蓝图？
- [ ] 角色参考图可用？
- [ ] 产品图已锁定（如适用）？
- [ ] 背景图已锁定（如适用）？
- [ ] 音乐风格和 BPM 已确定？
