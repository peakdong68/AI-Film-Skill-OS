# 制作状态机 — 核心知识

## 状态流转

```
STATE 0 → 输入采集       （收集简报，提问关键问题）
STATE 1 → 故事与情绪      （叙事结构 + 情绪弧线）
STATE 2 → 视觉设计        （摄影语法 + 灯光 + 色彩）
STATE 3 → 角色锁定        （角色设定表 + 身份锁定）
STATE 4 → 提示词封装      （电影级短片提示包编译：分镜设计 + 镜头语言 + 声音设计 + Part 分解方案）
    ↓
[路由决策：盘点资源 → 匹配 STATE 6 模式 → 选择方案]
    ↓                        ↓
STATE 5（条件性）         直通 STATE 6
分镜蓝图生成              （跳过 STATE 5）
    ↓                        ↓
    └────────┬───────────────┘
             ↓
STATE 6 → Seedance 视频提示词（多模式，图像引用级，平台可执行）
STATE 7 → 最终验证        （所有产物的质量审核）
STATE 8 → 导出就绪        （打包交付物）
```

STATE 0-4 为必选阶段，严禁跳过。STATE 5 为条件性阶段——仅当路由决策选择分镜蓝图方案（方案 A/C）时执行。STATE 6-8 为必选阶段。

## 绝对工作流锁

1. 故事结构和情绪弧线必须在视觉设计前确认
2. 摄影机语言和灯光系统必须在提示包编译前定义
3. 角色身份必须在提示包编译前确认
4. 提示包必须经用户确认后才能进入后续阶段
5. 如经 STATE 5，分镜蓝图图像必须在 Seedance 提示词前确认
6. 不假设确认——等待用户的明确批准
7. 提示包未确认 → 禁止进入下游阶段
8. 角色未确认 → Seedance 提示词禁止生成
9. STATE 4 输出严禁提及 Seedance/Kling 视频平台
10. STATE 6 必须根据可用资源选择正确模式——不可默认使用分镜驱动模式

## 阶段锁强制执行

| 锁 | 条件 | 违规后果 |
|----|------|---------|
| 故事锁 | 故事结构必须在视觉设计前确认 | 视觉语言不连贯 |
| 视觉锁 | 摄影机+灯光必须在提示包编译前定义 | 摄影机决策随机 |
| 角色锁 | 角色设定表必须在提示包编译前确认 | 角色漂移、换脸 |
| 提示包锁 | 提示包必须用户确认后才能进入后续阶段 | 下游方向错误 |
| 分镜锁 | 如经 STATE 5，分镜蓝图图像必须在 Seedance 提示词前确认 | 缺失叙事视觉参考 |
| 输出边界锁 | STATE 4 输出不得提及视频平台 | 用户误将文本提示词输入视频平台 |
| 提示词锁 | 所有预检项必须在最终导出前通过 | 连续性断裂 |
| 模式锁 | STATE 6 必须按可用资源选择模式，不可默认分镜驱动 | 提示词与输入不匹配、生成失败 |

## 路由决策规则（STATE 4 确认后）

提示包确认后，不直接进入 STATE 5。先执行三步路由决策：

### 第一步：盘点已有资源
询问用户持有哪些多模态资源（角色图、产品图、背景图、首尾帧、视频片段、音频片段）。

### 第二步：匹配 STATE 6 模式
| 已有资源 | 可选模式 |
|---|---|
| 无参考资源 | T2V |
| 单张参考图（产品/角色/场景） | I2V (minimal) |
| 首帧 + 尾帧 | FLF2V |
| 多种不同类型参考 | R2V |
| 视频源片段 | V2V Edit / V2V Extend |
| 需要分镜板控制多镜头连续运镜 | I2V (storyboard) → 需先经 STATE 5 |

### 第三步：给出 2-3 个方案
| 方案 | 路径 | 适用场景 |
|---|---|---|
| A: 分镜蓝图 | STATE 4 → STATE 5 → STATE 6 (I2V storyboard) | 叙事短片、复杂多镜头、需分镜板精确控制 |
| B: 直接生视频 | STATE 4 → STATE 6 (I2V minimal / FLF2V / T2V / R2V) | 已有参考图、产品展示、首尾帧过渡、纯文本 |
| C: 混合 | STATE 4 → STATE 5（关键镜头）+ STATE 6（并行） | 部分镜头需要分镜板，部分可直接用参考图 |

## STATE 4 与 STATE 5/6 的职责分离

**STATE 4（director-prompt-packager）**：文本级编译器
- 将 STATE 1-3 的所有设计产物编译为电影级短片提示包
- 包含：结构化分镜设计 + 镜头语言 + 声音设计 + Part 分解方案
- 供用户确认，作为后续阶段的设计基础
- **严禁在输出中提及 Seedance/Kling 视频平台**
- 输出为平台无关的导演级设计文档

**STATE 5（条件性，storyboard-sketch/storyboard-prompt/storyboard-master/storyboard-ecommerce）**：图像级生成
- 仅在路由决策方案 A/C 时执行
- 使用已确认的提示包生成分镜蓝图图像
- 生产中只需分镜蓝图板（总览图），而非逐帧独立图像
- 蓝图图像作为 STATE 6 I2V storyboard 模式的参考素材输入

**STATE 6（seedance-video-prompt）**：视频提示词编译器
- 支持 7 种模式，按可用资源自动选择
- 分镜图像仅在 I2V storyboard 模式下需要
- 每条提示词对应一个 Part（≤ 15s），独立计数

## 多 Part 结构规则

- 每个 Part = 一个 Seedance 2.0 调用 = 一个视频片段（最长 15 秒）
- 每个 Part 独立编写一条提示词，独立受字数限制约束（中文 ≤ 500 字 / 英文 ≤ 1000 词）
- 总时长 = Part 数量 × 每 Part 秒数
- Part 1：由所选模式驱动（分镜图 / 参考图 / 首尾帧 / 纯文本）
- Part 2+：必须引用前一个 Part 的输出视频作为连续性基线（`` `视频1` 为前一个 Part 的输出``）
- 多 Part 项目在 STATE 6 逐 Part 编译提示词，非一次性输出全部

## 预检清单（STATE 4 提示包编译前）

全部必须是"是"：
- [ ] 故事结构已确认？
- [ ] 情绪弧线已确认？
- [ ] 视觉语言（摄影机 + 灯光）已定义？
- [ ] 角色身份已锁定并确认？
- [ ] 时长和画幅比例已锁定？
- [ ] Part 分解方案已确定（每个 Part ≤ 15s）？

## 预检清单（STATE 6 Seedance 提示词生成前，按模式验证）

- [ ] 路由决策已完成，方案已选定？
- [ ] 所选模式的必需输入已就绪？
- [ ] 如方案 A/C：分镜蓝图图像已生成且用户已确认？
- [ ] 如方案 B：参考图（单张/首尾帧/视频片段）已就绪？
- [ ] 角色参考图可用？（或用户已明确选择跳过）
- [ ] 产品图已锁定（如适用）？
- [ ] 背景图已锁定（如适用）？
- [ ] 音乐风格和 BPM 已确定？
- [ ] 每条提示词字数合规（中文 ≤ 500 字 / Part）？
