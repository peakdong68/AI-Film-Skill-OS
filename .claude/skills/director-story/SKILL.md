# Director Story — 剧本导演引擎

## 概览

将故事创意或剧本转化为专业的叙事蓝图。此技能应用导演级分析：将故事拆分为幕结构、提取场景目的、构建因果链，并定义每场戏必须在叙事上达成什么。产出是所有下游系统（情绪、摄影机、分镜、提示词）赖以构建的结构基础。

此技能可独立用于剧本分析，或由 `director-core` 在 STATE 1 调用。

## 输入门

接受以下任意作为起始材料：
- 故事概念（一句到一段）
- 剧本或剧本节选
- 节拍大纲或 treatment
- 场景列表
- 主题概念加角色描述

如果用户提供的素材不足 20 个字，先追问更多细节再继续。如果用户说"直接生成"，从已有材料扩展并标注假设。

## 输出结构

每次分析必须产出一个包含以下 5 部分的剧本导演蓝图：

### 1. 叙事结构图

选择一个结构并填充：

**3幕结构:**
```
Act 1: 建置 — [世界、角色、现状]
  转折点 1: [激励事件]
Act 2: 对抗 — [上升的障碍、复杂化]
  中点转变: [不可逆的变化]
  转折点 2: [最黑暗的时刻 / 危机]
Act 3: 解决 — [高潮、新平衡]
```

**5幕结构:**
```
Act 1:  exposition — [世界与角色建立]
Act 2: 上升行动 — [复杂化累积]
Act 3: 高潮 — [冲突顶点、决定性时刻]
Act 4: 下降行动 — [后果展开]
Act 5: 收场 — [解决、新状态]
```

短内容（≤30s）默认使用 3幕。长内容（>60s）考虑 5幕。

### 2. 场景拆解图

为每场戏定义：

```
SCENE [N]: [场景标题]
- 场景目的: [这场戏在叙事中为什么存在]
- 情绪功能: [观众感受到什么情绪？]
- 叙事功能: [推进剧情 / 揭示角色 / 建立世界 / 制造紧张 / 解决]
- 涉及角色: [谁出现]
- 冲突类型: [内在 / 人际 / 外部 / 环境]
- 进入状态 → 退出状态: [这场戏结束时什么变了？]
```

### 3. 因果链

定义驱动叙事前进的因果链条：

```
事件 A → 触发 → 事件 B → 导致 → 事件 C
```

映射依赖关系：
```
[场景1事件] → 导致 → [场景2情境] → 迫使 → [场景3决定] → 造成 → [高潮]
```

对每个链接，验证："如果我删除这个事件，下一场戏还成立吗？"如果成立，因果链接就弱。

### 4. 导演意图层

为每场戏回答三个导演问题：

| 问题 | 揭示什么 |
|------|---------|
| 这场戏为什么存在？ | 叙事理由——如果删掉它什么都不会丢失，那它就不该存在 |
| 这场戏之后什么变了？ | 增量——观众认知、角色状态、紧张程度 |
| 观众带着什么情绪离开？ | 情绪残留——什么感受会延续到下一场戏 |

### 5. 分镜准备度检查

评估叙事结构是否已准备好进入分镜：

- [ ] 每场戏都有明确的目的
- [ ] 因果链没有断裂
- [ ] 情绪弧线覆盖完整时长
- [ ] 角色动机驱动剧情，而非巧合
- [ ] 每场戏的导演意图清晰

如果有任何项未勾选，在移交分镜前注明缺失了什么。

## 场景目的分类

使用以下分类来定义场景目的。每场戏必须至少属于其中一类：

| Category | 中文 | 信号 |
|----------|------|------|
| Establish | 建立 | 引入世界、角色或现状 |
| Advance | 推进 | 通过行动或决定推动剧情前进 |
| Reveal | 揭示 | 暴露信息、角色真相或赌注 |
| Escalate | 升级 | 增加紧张、赌注或冲突 |
| Contrast | 对比 | 并列两种状态、角色或想法 |
| Transition | 过渡 | 连接主要叙事区块 |
| Resolve | 收束 | 结束一段弧线、回答一个问题、达成闭合 |

## 约束

- 没有无目的的场景——如果一场戏仅因美学原因存在，标记它并让用户确认。
- 没有孤立事件——每个事件必须连接到因果链。
- 没有未经审视的导演意图——每场戏必须通过三个问题。
- 在准备度检查通过之前，不移交到分镜。

## 与 director-core 的集成

当被 `director-core` 在 STATE 1 调用时：
- 产出完整的剧本导演蓝图
- 呈现给用户确认
- 确认后，标记 STATE 1 完成
- 蓝图输入到 `director-emotion` 用于情绪时间轴
