# Director Story — 剧本导演引擎

## 概览

将故事创意或剧本转化为专业的叙事蓝图。此技能应用导演级分析：将故事拆解为幕，提取场景目的，构建因果链，并定义每个场景必须在叙事上达成的目标。输出是下游所有系统（情绪、摄影机、分镜、提示词）所依赖的结构性基础。

此技能可独立用于剧本分析，也可被 `director-core` 在 STATE 1 调用。

## 加载资源

此技能附带参考知识文件。在以下情况下加载：

- 获取场景目的分类法和完整叙事结构模板，阅读 `references/script-breakdown.md`

## 输入门控

接受以下任意一种作为起始素材：

- 一个故事概念（一句话到一段话）
- 一份剧本或剧本片段
- 一份节拍大纲或导演阐述
- 一个场景列表
- 一个主题概念及角色描述

**模糊或描述充分但缺剧情的输入，路由到 `director-interview` 进行创意入站访谈。** 该技能自动判断并选择最佳路径：

| 输入质量         | 路径                   |
| ---------------- | ---------------------- |
| 创意完整         | 快速通过，直接分析     |
| 描述充分但缺剧情 | 2-3 方案供选择后再分析 |
| 创意模糊         | 逐项访谈后再分析       |

如果用户明确要求“直接分析”且拒绝访谈，基于已有素材展开并标注假设。

## 输出结构

每次分析必须产出一份剧本导演蓝图，包含以下 5 个部分：

### 1. 叙事结构图

选择并填充一种结构：

**三幕结构：**

```
第一幕：铺垫 — [世界观、角色、现状]
  转折点 1：[激励事件]
第二幕：对抗 — [逐渐升级的障碍与复杂化]
  中点转折：[不可逆的变化]
  转折点 2：[至暗时刻 / 危机]
第三幕：解决 — [高潮、新平衡]
```

**五幕结构：**

```
第一幕：展示 — [建立世界观与角色]
第二幕：上升 — [冲突累积]
第三幕：高潮 — [冲突峰值、决定性时刻]
第四幕：下落 — [后果展开]
第五幕：收束 — [解决、新状态]
```

短内容（≤30 秒）默认使用三幕。长内容（>60 秒）考虑五幕。

### 2. 场景拆解图

对每个场景进行定义：

```
场景[N]：[场景标题]
- 场景目的：[为什么此场景存在于叙事中]
- 情绪功能：[观众感受到什么情绪？]
- 叙事功能：[推进情节 / 揭示角色 / 构建世界 / 制造张力 / 解决]
- 涉及角色：[谁出现]
- 冲突类型：[内在 / 人际 / 外部 / 环境]
- 入场状态 → 离场状态：[场景结束时改变了什么？]
```

### 3. 因果链

定义驱动叙事前进的因果链条：

```
事件 A → 触发 → 事件 B → 后果 → 事件 C
```

映射依赖关系：

```
[场景 1 事件] → 导致 → [场景 2 情境] → 迫使 → [场景 3 决策] → 结果 → [高潮]
```

对每个环节验证："如果移除这个事件，下一个场景还成立吗？"如果是——因果链接薄弱。

### 4. 导演意图层

对每个场景回答三个导演问题：

| 问题                       | 揭示什么                                             |
| -------------------------- | ---------------------------------------------------- |
| 这个场景为什么存在？       | 叙事正当性——如果删除场景不损失任何东西，它就不该存在 |
| 这个场景结束后改变了什么？ | 变化量——观众认知、角色状态、张力水平                 |
| 观众带着什么情绪离开？     | 情绪残留——什么感受带入下一个场景                     |

### 5. 分镜准备度检查

评估叙事结构是否已为分镜规划做好准备：

- [ ] 每个场景都有清晰的目的
- [ ] 因果链没有断点
- [ ] 情绪弧线覆盖全片时长
- [ ] 角色动机驱动情节，而非偶然推动
- [ ] 每个场景的导演意图清晰

如果有任何一项未勾选，在交接给分镜前标注缺失内容。

## 场景目的分类法

用以下类别分类场景目的。每个场景必须至少属于其中一类：

| 类别 | 信号                             |
| ---- | -------------------------------- |
| 建立 | 引入世界观、角色或现状           |
| 推进 | 通过行动或决策推动情节           |
| 揭示 | 曝光信息、角色真相或利害关系     |
| 升级 | 增加张力、赌注或冲突             |
| 对比 | 并置两种状态、角色或理念         |
| 过渡 | 连接主要叙事块之间的桥梁         |
| 收束 | 完成一个弧线、回答问题、达成闭合 |

## 约束

- 没有无目的的场景——如果场景仅出于美学原因存在，标注并要求用户确认。
- 没有孤立的事件——每个事件必须连接到因果链。
- 没有未经审视的导演意图——每个场景必须通过三个问题。
- 准备度检查通过之前不交接给分镜。

## 与 director-core 的集成

被 `director-core` 在 STATE 1 调用时：

- 产出完整的剧本导演蓝图
- 呈现给用户确认
- 确认后，标记 STATE 1 完成
- 蓝图输入 `director-emotion` 以生成情绪时间线
