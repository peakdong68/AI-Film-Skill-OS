# Seedance Video Prompt — L5 视频生成编译器

## 概览

AI Film OS 管线中 **L5 — 视频生成层** 的最终编译器。接收多模态参考，编译为 Seedance 2.0 / Kling 平台可执行的视频提示词。

**关键区分：**

- STATE 4（`director-prompt-packager`）：文本级编译器。**不是视频平台提示词。**
- STATE 6（`seedance-video-prompt`）：图像引用级编译器 → 平台可执行提示词。

**分镜帧图像不是必须的**——仅在 storyboard 驱动模式下需要。

## 输出结构模板

**所有模式的所有提示词必须遵循此骨架。** 模式特定差异仅体现在必选/可选字段上。

### 标准 Prompt 骨架

```
[素材声明段]          ← 必选（有参考素材时），无参考素材（T2V）时省略
[主体定义段]          ← 必选（多角色/有参考图时），单主体可简化
[分镜蓝图约束段]      ← 必选（I2V storyboard 模式），其他模式省略
[叙事内容段]          ← 必选：主体 + 场景 + 动作序列
[镜头与运镜段]        ← 必选：一个主要运镜方式
[光影与色彩段]        ← 可选：物理光源、色温、氛围
[音频段]              ← 可选：音乐 ()、音效 <>、台词 {}
[约束段]              ← 必选：保留项 + 排除项
```

### 各段规范

| 段               | 必选条件                                       | 写法要点                                                                             |
| ---------------- | ---------------------------------------------- | ------------------------------------------------------------------------------------ |
| 素材声明         | 有 `@[图片N]` / `@[视频N]` / `@[音频N]` 素材时 | 声明每个素材的角色（身份/动作/场景/节奏）                                            |
| 主体定义         | 有角色参考图时                                 | `将@[图片N]中[2-3个静态特征]定义为`主体N``                                           |
| **分镜蓝图约束** | **I2V storyboard 模式**                        | **必须写明：不渲染分镜表本身、忽略面板边框和文字标签、严格遵循面板顺序和摄像机角度** |
| 叙事内容         | 始终                                           | 按分镜面板顺序逐镜头描述，每镜头=一个可见节拍                                        |
| 镜头运镜         | 始终                                           | 一个主要运镜及终点，禁止多运镜堆叠                                                   |
| 光影色彩         | 有特定光源/色温时                              | 物理光源+方向+色温，禁止空洞形容词                                                   |
| 音频             | 有音乐/音效/对白时                             | 音乐用（）、音效用<>、台词用{}                                                       |
| 约束             | 始终                                           | `无字幕、无Logo、无水印` 为基础；I2V追加 `无面部变形、无身份漂移`                    |

### I2V storyboard 分镜蓝图约束（硬性规则）

**当提示词中包含分镜参考图（`@[图片N]` 为分镜蓝图）时，必须包含以下完整约束，不可简化为"参考分镜图"一句话：**

```
将@[图片N]分镜图作为视频叙事蓝图，不渲染分镜表本身，
忽略面板边框和文字标签，
严格遵循面板顺序、摄像机角度和动作节奏。
```

**约束语义说明：**

| 约束项                 | 含义                            | 为何必须                               |
| ---------------------- | ------------------------------- | -------------------------------------- |
| 作为视频叙事蓝图       | 分镜图定义镜头序列和叙事结构    | 告诉模型分镜图是"剧本"不是"素材"       |
| 不渲染分镜表本身       | 输出是视频，不是分镜板图像      | 防止模型把分镜板当作画面内容直接输出   |
| 忽略面板边框和文字标签 | 排除分镜板UI元素                | 防止模型渲染表格线条、镜头编号、时间码 |
| 严格遵循面板顺序       | 按01→02→03...顺序生成           | 保证叙事因果链不被模型打乱             |
| 严格遵循摄像机角度     | 每个面板指定的景别/角度必须对应 | 保证导演的镜头设计被忠实执行           |
| 严格遵循动作节奏       | 面板之间的时间分配和动作密度    | 保证情绪弧线不被模型自由发挥           |

## 模式门控

Seedance 2.0 官方定义了**三大任务类型**。每个大类下覆盖若干子模式，按可用资源选择。

| 任务类型       | 子模式                                           | 必需输入                                 | 核心写法                                         |
| -------------- | ------------------------------------------------ | ---------------------------------------- | ------------------------------------------------ |
| **多模态参考** | T2V / I2V minimal / I2V storyboard / R2V / FLF2V | 至少一种参考（图片/视频/音频），或纯文本 | 从参考中提取元素，生成全新视频                   |
| **编辑视频**   | V2V Edit                                         | `@[视频1]`                               | 在原视频基础上局部/全局修改，**不加"参考"前缀**  |
| **延长视频**   | V2V Extend                                       | `@[视频1]`                               | 时间维度延续，音视频风格一致，**不加"参考"前缀** |

### 模式判定树

**第一步：检查路由锁定。** 当由 `director-core` STATE 6 调用时，检查上游 STATE 4 的路由决策。若路由已锁定方案，直接使用对应模式，不重新判定可用素材。

| 路由锁定                 | 强制模式         | 说明                                                          |
| ------------------------ | ---------------- | ------------------------------------------------------------- |
| 方案 A（I2V storyboard） | I2V (storyboard) | 即使用户尚未生成素材，也按 I2V 格式产出 `@[图片N]` 占位提示词 |
| 方案 B（直通 STATE 6）   | 按素材判定       | 沿用第二步判定逻辑                                            |

> **关键规则：方案 A 锁定后，STATE 6 必须使用 I2V (storyboard) 模式编写提示词。**
> 素材可能尚未生成——此时使用 `@[图片N]` 占位符，并在提示词前附带"素材映射（上传后生效）"说明。

**第二步：无路由锁定时，按素材判定（现有逻辑）。**

```
有视频源需修改？ → 编辑视频（直接 @[视频1]，不加"参考"）
有视频需延续？ → 延长视频（直接 @[视频1]，不加"参考"）
有参考图？ → I2V（storyboard）— 主要推荐
仅单张参考图？ → I2V（minimal）— 简化备选
有首帧+尾帧？ → FLF2V
有多种不同类型参考？ → R2V
仅有文字描述？ → T2V

> **对于 I2V，I2V（storyboard）是主要推荐。** 它通过分镜蓝图图像提供最佳的多镜头控制。I2V（minimal）是单张参考图场景的简化选项。始终向用户展示两种选项及其权衡。
```

> 详细模板与范例见 `references/seedance-methodology.md`

### 占位素材声明（I2V 模式素材未生成时使用）

当 I2V 模式下素材尚未生成时，提示词前必须附带素材映射声明：

```
素材映射（上传对应素材后生效）：
| 上传序号 | 素材 | 来源 |
|----------|------|------|
| `@[图片1]` | [角色A] Character Sheet | character-sheets.md |
| `@[图片2]` | [角色B] Character Sheet | character-sheets.md |
| `@[图片3]` | 分镜总览图 Master Sheet | State-5-storyboard.md |
```

> 此表让用户明确知道需要上传哪些素材、按什么顺序上传。

## 导演公式

```
精准主体 + 动作细节 + 场景环境 + 光影色调 + 镜头运镜 + 视觉风格 + 画质 + 约束条件
```

| 槽位      | 用途                          | 范例                                          |
| --------- | ----------------------------- | --------------------------------------------- |
| 主体      | 模型追踪的锚点，优先定义      | `将@[图片1]中穿红色连衣裙的女人定义为`主体1`` |
| 动作      | 可见变化，需肢体细化+程度量化 | `缓慢抬手、轻轻点头、借着转身惯性顺势坐下`    |
| 场景      | 仅写参考中未显示的内容        | `安静的雨光洒落的厨房台面，浅景深`            |
| 镜头      | 一个主要运镜及终点            | `中景平稳跟拍，缓慢推近至特写`                |
| 光影/风格 | 物理光源 + 视觉调性           | `暖黄日光从窗外洒进，冷蓝轮廓光，电影纪实风`  |
| 画质      | 清晰度、细节纹理              | `高清，细节丰富，电影质感，色彩自然`          |
| 音频      | 音效/对白/音乐                | `（背景中播放快节奏摇滚乐）<远处传来狗叫声>`  |
| 约束      | 保留项和排除项                | `保持无字幕、不要生成Logo、不要生成水印`      |

## 参考格式

Seedance 2.0 使用 `@[图片N]` / `@[视频N]` / `@[音频N]` 格式指代上传素材。素材须按顺序上传，提示词中用序号准确指代。

| 素材类型 | 写法                  | 用途                                      |
| -------- | --------------------- | ----------------------------------------- |
| 图片     | `@[图片1]` `@[图片2]` | 角色锚定、场景定调、产品锁定、首帧尾帧    |
| 视频     | `@[视频1]`            | 运镜参考、动作参考、特效参考、编辑/延长源 |
| 音频     | `@[音频1]`            | 节奏氛围、音色参考                        |

### 主体定义规则

必须明确定义参考素材中的主体，确保唯一可识别性。

**核心原则：参考图本身就是身份定义——不要逐项复述参考图中已包含的静态外貌。** 复述面部结构、发型、体型等已在图像中锁定的细节，会导致模型重新解读而非直接引用参考图。主体定义仅需 1-2 个显著外观标识，其余由参考图承载。

**有参考图时（I2V / R2V / FLF2V）：**

| 场景             | 写法                                                        | 示例                                                       |
| ---------------- | ----------------------------------------------------------- | ---------------------------------------------------------- |
| **基本定义**     | `` 将@[图片1]中的[1-2个显著外观标识]定义为`主体1` ``        | `` 将@[图片1]中的男性模特定义为`模特` ``                   |
| **简化绑定**     | `` `主体1`@[图片1] ``                                       | `` `模特`@[图片1] ``                                       |
| **多素材同主体** | `` 将@[图片1]中的[标识]、@[图片2]中的[标识]定义为`主体N` `` | `` 将@[图片1]中的角色面容、@[图片2]中的服装定义为`主角` `` |
| **面部特写分离** | `` `主体1`的面部参考@[图片1]，服装参考@[图片2] ``           | 大头照+全身照分离引用，增强面部提取权重                    |

**无参考图时（T2V）：** 需在提示词中详细描述主体外观（面部+发型+体型+服装）。

**反模式（有参考图时禁止）：**

- ❌ `` 将@[图片1]中亚洲男性模特的面部结构、韩式微卷深色短发、精瘦运动体型、白色印花圆领短袖T桖定义为`模特` `` — 过度描述了已在参考图中锁定的细节
- ✅ `` 将@[图片1]中的男性模特定义为`模特` `` — 仅作身份绑定，不重新描述

### 编辑/延长视频规则

编辑和延长任务**严禁使用"参考"前缀**，直接使用 `` `视频N` `` 指代，以免被误判为参考任务。

- ✅ `编辑@[视频1]，将其中的[原特征]修改为[新特征]`
- ✅ `向后延长@[视频1]，生成...`
- ❌ `参考@[视频1]，编辑...`

## 动作描述规则

**1. 肢体细化 + 程度量化**：动作需具体到手、腿、头部等肢体部位，补充幅度、速度、力度。示例：`缓慢抬手、快速转头、用力蹬地、微微低头`。

**2. 优先低缓连续小动作**：选用缓慢、轻柔、连贯的细微动作，规避狂奔、大跳、剧烈翻滚等高爆发大动态。示例：`缓慢行走、轻轻抬手、微微低头、顺势坐下`。

**3. 补充动作过渡衔接**：写明前后动作的惯性与承接关系。示例：`借着转身惯性顺势抬手、从停顿状态自然过渡到举手`。

**4. 情绪具象外化**：用身体细节替代抽象情绪词。

| 抽象情绪  | 外化为动作与细节                                     |
| --------- | ---------------------------------------------------- |
| 悲伤      | 低头、肩膀微微颤抖、眼眶泛红、手指无意识地攥紧衣角   |
| 喜悦      | 嘴角抑制不住地上扬、眉眼舒展、脚步变得轻快           |
| 紧张/焦虑 | 频繁看手表、手指不停敲击桌面、呼吸急促、眼神闪躲     |
| 愤怒      | 双拳紧握、下颌线紧绷、胸口剧烈起伏、眼神如刀般锐利   |
| 释然      | 长长舒了一口气、紧绷的肩膀完全放松、脸上露出淡淡微笑 |

## 特殊字符规范

| 用途 | 符号   | 示例                                                     |
| ---- | ------ | -------------------------------------------------------- |
| 音乐 | `（）` | `（背景中播放着快节奏的摇滚乐）`                         |
| 音效 | `<>`   | `<远处传来狗叫声>`                                       |
| 台词 | `{}`   | `{你好，世界}`。小语种标注语种：`用日语说道{こんにちは}` |
| 字幕 | `【】` | `【第一章：启程】`                                       |

## 分模式提示词规范

以下为各模式的提示词骨架和关键约束。详细模板、故障修复、场景适配见 `references/seedance-methodology.md`。

### I2V 精确保留（minimal — 单张参考图）

```
@[图片1]为参考；精确保留[身份/产品/场景]。仅[运动]发生变化。
镜头：[单一运镜]。光影：[光源或过渡]。音效：[提示音]。约束：[不变项]。
```

适用：仅单张角色图/产品图，无分镜蓝图。重点：静态特征锁定，仅描述运动。

### I2V 分镜驱动（storyboard — 多镜头蓝图）

```
将@[图片N]分镜图作为视频叙事蓝图，不渲染分镜表本身，
忽略面板边框和文字标签，
严格遵循面板顺序、摄像机角度和动作节奏。

将@[图片1]中[特征]定义为`主体1`。
将@[图片2]中[特征]定义为`主体2`。

镜头1（面板01对应——[叙事目的]）：[场景]，[主体动作]。[景别]，[运镜]。
镜头2（面板02对应——[叙事目的]）：[场景]，[主体动作]。[景别]，[运镜]。
...

[光影/色彩]
（[音乐描述]）<[音效]> {[台词]}

约束：无字幕，无Logo，无水印。无面部变形，无身份漂移。
```

适用：有分镜蓝图图像 + 角色参考图。重点：逐面板镜头映射 + 完整分镜蓝图约束。

### R2V 多参考角色映射

```
将@[图片1]中的[特征]定义为`主体1`。
使用@[视频1]仅控制运镜节奏，不复制表演者、房间、品牌或服装。
使用@[音频1]仅控制节奏和能量。

`主体1`[动作]，[场景]。镜头：[运镜]。音效：[SFX]。
约束：[不变项]。
```

适用：多种不同类型参考素材混用。重点：每种素材分配单一角色，声明"不传递项"。

### FLF2V 首帧/尾帧


## 可见节拍原则

**每段剪辑只放一个可见节拍。** 节拍类型：揭示、登场、决定、变形、接触、追逐、消失。一个节拍 = 一个动作动词 + 可见终点。不要在一个片段里塞入多个事件转折。

## I2V 核心原则

**只描述图像无法传达的内容。** 静态图像已含主体身份、产品形态、构图。重新描述静态细节会导致漂移。仅添加：运动、镜头、光影变化、声音、保留约束。

| 正确的添加 | 示例                                   |
| ---------- | -------------------------------------- |
| 微表情     | `主体眨一次眼，视线微微下移`           |
| 产品扫光   | `一条细高光扫过标签表面`               |
| 天气/氛围  | `雨丝在身后划过，灰尘在门口光束中扬起` |
| 镜头运动   | `从当前构图缓慢推近至更紧凑的细节`     |

## Anti-Slop

用可被拍摄/录制的制作语言替换空洞评价词。核心：cinematic → 景别+运镜+布光+调色；epic → 物理尺度+镜头距离；beautiful → 颜色+质感+光线行为。

> 完整替换词库（13 种空洞词 → 制作语言对照）见 `../references/anti-slop-lexicon.md`

**规则：如果摄影机、麦克风、测光表或秒表无法检测到它——重写。**

## 验证清单

- [ ] 模式已声明且与可用输入匹配
- [ ] 参考素材使用 `@[图片N]` / `@[视频N]` / `@[音频N]` 格式
- [ ] 主体已明确定义（`` 将@[图片N]中的[特征]定义为`主体N` ``）
- [ ] 编辑/延长模式：未使用"参考"前缀，直接 `` `视频N` ``
- [ ] 动作含肢体细化+程度量化，优先低缓连续小动作
- [ ] 情绪用身体细节表达，无抽象情绪词
- [ ] 音效/对白/音乐使用了正确特殊字符（`（）<>{}`）
- [ ] 一个镜头仅指定一种运镜方式
- [ ] 字数合规：中文 ≤ 500 字（已标注计数）
- [ ] Anti-slop 检查通过
- [ ] 约束词完整：`保持无字幕、不要生成Logo、不要生成水印`

## 与 director-core 集成

STATE 6 调用时：

1. 检查可用输入，不假设分镜图存在
2. 按模式判定树选择模式
3. 主体定义：将参考图中的人物/产品明确定义为主体标签
4. 按所选模式编译可执行提示词
5. 字数计数，执行验证清单
6. 提交用户审核，确认后标记 STATE 6 完成
