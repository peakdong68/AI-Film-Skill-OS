# Seedance Video Prompt — L5 视频生成编译器

## 概览

AI Film OS 管线中 **L5 — 视频生成层** 的最终编译器。接收多模态参考，编译为 Seedance 2.0 / Kling 平台可执行的视频提示词。

**关键区分：**
- STATE 4（`director-prompt-packager`）：文本级编译器。**不是视频平台提示词。**
- STATE 6（`seedance-video-prompt`）：图像引用级编译器 → 平台可执行提示词。

**分镜帧图像不是必须的**——仅在 storyboard 驱动模式下需要。

## 模式门控

Seedance 2.0 官方定义了**三大任务类型**。每个大类下覆盖若干子模式，按可用资源选择。

| 任务类型 | 子模式 | 必需输入 | 核心写法 |
|---|---|---|---|
| **多模态参考** | T2V / I2V minimal / I2V storyboard / R2V / FLF2V | 至少一种参考（图片/视频/音频），或纯文本 | 从参考中提取元素，生成全新视频 |
| **编辑视频** | V2V Edit | `` `视频1` `` | 在原视频基础上局部/全局修改，**不加"参考"前缀** |
| **延长视频** | V2V Extend | `` `视频1` `` | 时间维度延续，音视频风格一致，**不加"参考"前缀** |

### 模式判定树

```
有视频源需修改？ → 编辑视频（直接 `视频1`，不加"参考"）
有视频需延续？ → 延长视频（直接 `视频1`，不加"参考"）
有分镜图？ → I2V storyboard
有首帧+尾帧？ → FLF2V
有多种不同类型参考？ → R2V
有单张参考图？ → I2V minimal
仅有文字描述？ → T2V
```

> 详细模板与范例见 `references/seedance-methodology.md`

## 导演公式

```
精准主体 + 动作细节 + 场景环境 + 光影色调 + 镜头运镜 + 视觉风格 + 画质 + 约束条件
```

| 槽位 | 用途 | 范例 |
|---|---|---|
| 主体 | 模型追踪的锚点，优先定义 | `将`图片1`中穿红色连衣裙的女人定义为`主体1`` |
| 动作 | 可见变化，需肢体细化+程度量化 | `缓慢抬手、轻轻点头、借着转身惯性顺势坐下` |
| 场景 | 仅写参考中未显示的内容 | `安静的雨光洒落的厨房台面，浅景深` |
| 镜头 | 一个主要运镜及终点 | `中景平稳跟拍，缓慢推近至特写` |
| 光影/风格 | 物理光源 + 视觉调性 | `暖黄日光从窗外洒进，冷蓝轮廓光，电影纪实风` |
| 画质 | 清晰度、细节纹理 | `高清，细节丰富，电影质感，色彩自然` |
| 音频 | 音效/对白/音乐 | `（背景中播放快节奏摇滚乐）<远处传来狗叫声>` |
| 约束 | 保留项和排除项 | `保持无字幕、不要生成Logo、不要生成水印` |

## 参考格式

Seedance 2.0 使用 `` `图片N` `` / `` `视频N` `` / `` `音频N` `` 格式指代上传素材。素材须按顺序上传，提示词中用序号准确指代。

| 素材类型 | 写法 | 用途 |
|---|---|---|
| 图片 | `` `图片1` `` `` `图片2` `` | 角色锚定、场景定调、产品锁定、首帧尾帧 |
| 视频 | `` `视频1` `` | 运镜参考、动作参考、特效参考、编辑/延长源 |
| 音频 | `` `音频1` `` | 节奏氛围、音色参考 |

### 主体定义规则

必须明确定义参考素材中的主体，确保唯一可识别性。使用 2–3 个清晰稳定的静态特征（服饰、发型、外观类别）描述。

**基本定义：** `` 将`图片1`中[特征1]、[特征2]定义为`主体1` ``
**简化绑定：** `` `主体1`@`图片1` `` / `` 张三@图片1 ``
**多素材同主体：** `` 将`图片1`中的[...]、`图片2`中的[...]定义为`主体N` ``
**面部特写分离：** `` `主体1`的面部特征参考`图片1`（大头照），妆造参考`图片2`（全身照） ``
（可选——当身份一致性要求较高时推荐使用，有助于提升 Seedance 对面部特征的提取权重）

### 编辑/延长视频规则

编辑和延长任务**严禁使用"参考"前缀**，直接使用 `` `视频N` `` 指代，以免被误判为参考任务。

- ✅ `` 编辑`视频1`，将其中的[原特征]修改为[新特征] ``
- ✅ `` 向后延长`视频1`，生成... ``
- ❌ `` 参考`视频1`，编辑... ``

## 动作描述规则

**1. 肢体细化 + 程度量化**：动作需具体到手、腿、头部等肢体部位，补充幅度、速度、力度。示例：`缓慢抬手、快速转头、用力蹬地、微微低头`。

**2. 优先低缓连续小动作**：选用缓慢、轻柔、连贯的细微动作，规避狂奔、大跳、剧烈翻滚等高爆发大动态。示例：`缓慢行走、轻轻抬手、微微低头、顺势坐下`。

**3. 补充动作过渡衔接**：写明前后动作的惯性与承接关系。示例：`借着转身惯性顺势抬手、从停顿状态自然过渡到举手`。

**4. 情绪具象外化**：用身体细节替代抽象情绪词。

| 抽象情绪 | 外化为动作与细节 |
|---|---|
| 悲伤 | 低头、肩膀微微颤抖、眼眶泛红、手指无意识地攥紧衣角 |
| 喜悦 | 嘴角抑制不住地上扬、眉眼舒展、脚步变得轻快 |
| 紧张/焦虑 | 频繁看手表、手指不停敲击桌面、呼吸急促、眼神闪躲 |
| 愤怒 | 双拳紧握、下颌线紧绷、胸口剧烈起伏、眼神如刀般锐利 |
| 释然 | 长长舒了一口气、紧绷的肩膀完全放松、脸上露出淡淡微笑 |

## 特殊字符规范

| 用途 | 符号 | 示例 |
|---|---|---|
| 音乐 | `（）` | `（背景中播放着快节奏的摇滚乐）` |
| 音效 | `<>` | `<远处传来狗叫声>` |
| 台词 | `{}` | `{你好，世界}`。小语种标注语种：`用日语说道{こんにちは}` |
| 字幕 | `【】` | `【第一章：启程】` |

## 平台硬约束

每条 Seedance 2.0 提示词对应一个 Part（≤ 15s），独立计数。

| 语言 | 限制（每条） |
|---|---|
| **中文** | ≤ 500 字 |
| **英文** | ≤ 1000 词 |
| **总字符数** | ≤ 2000 字符 |

多 Part 项目每个 Part 独立编写一条提示词。Part 2+ 需引用前一 Part 输出视频作为连续性参考（`` 向后延长`视频1`，保持... ``）。

## 核心模板

### 多模态参考 — I2V 精确保留

```
`图片1`为参考；精确保留[身份/产品/场景]。仅[运动]发生变化。
镜头：[单一运镜]。光影：[光源或过渡]。音效：[提示音]。约束：[不变项]。
```

### 多模态参考 — 多角色 R2V

```
将`图片1`中的[特征1]定义为`主体1`，将`图片2`中的[特征2]定义为`主体2`。
`主体1`[动作描述]；`主体2`[动作描述]。场景：[环境]。
镜头：[运镜]。风格：[调性]。约束：保持无字幕、不要生成Logo。
```

### 编辑视频 — V2V Edit

```
编辑`视频1`，将其中的[原特征]修改为[新特征]。未提及的部分默认保持不变。
```

### 延长视频 — V2V Extend

```
向后延长`视频1`，[主体]继续[动作]。音视频风格、主体与叙事保持一致。
```

> 完整模板与场景适配见 `references/seedance-methodology.md`

## 可见节拍原则

**每段剪辑只放一个可见节拍。** 节拍类型：揭示、登场、决定、变形、接触、追逐、消失。一个节拍 = 一个动作动词 + 可见终点。不要在一个片段里塞入多个事件转折。

## I2V 核心原则

**只描述图像无法传达的内容。** 静态图像已含主体身份、产品形态、构图。重新描述静态细节会导致漂移。仅添加：运动、镜头、光影变化、声音、保留约束。

| 正确的添加 | 示例 |
|---|---|
| 微表情 | `主体眨一次眼，视线微微下移` |
| 产品扫光 | `一条细高光扫过标签表面` |
| 天气/氛围 | `雨丝在身后划过，灰尘在门口光束中扬起` |
| 镜头运动 | `从当前构图缓慢推近至更紧凑的细节` |

## Anti-Slop

替换空洞评价词为可被拍摄/录制的制作语言。

| 空洞词 | 替换为 |
|---|---|
| cinematic/电影感 | 景别 + 运镜 + 布光 + 调色 |
| epic/史诗感 | 物理尺度 + 镜头距离 |
| beautiful/唯美 | 颜色 + 质感 + 构图 + 光线行为 |
| dynamic/动感 | 具体运动 + 速度 + 终点 |
| dramatic/戏剧化 | 站位调度 + 阴影 + 静默 + 镜头压力 |

**规则：如果摄影机、麦克风、测光表或秒表无法检测到它——重写。**

## 与 director-core 集成

STATE 6 调用时：

1. 检查可用输入，不假设分镜图存在
2. 按模式判定树选择模式
3. 主体定义：将参考图中的人物/产品明确定义为主体标签
4. 按所选模式编译可执行提示词
5. 字数计数，执行验证清单
6. 提交用户审核，确认后标记 STATE 6 完成
