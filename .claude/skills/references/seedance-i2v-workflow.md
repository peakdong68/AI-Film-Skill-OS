# Seedance I2V 工作流 — 内置参考知识

为 `storyboard-sketch` 技能提供 Seedance 图生视频（I2V）特定的分镜规划知识。

## Seedance 操作模式

| 模式 | 编译优先级 | 常见错误 | 修复 |
|------|---------|---------|------|
| **T2V** | 用紧凑层构建完整镜头 | 一个片段事件过多 | 保持一个可见节拍+一个终点 |
| **I2V** | 保留可见身份；添加运动 | 重新描述图像直到产品或面部漂移 | 写 `严格保留 @[图片1]`；仅添加动态变化 |
| **R2V** | 为每个资源分配独立角色 | 一个资源被要求同时控制身份、姿势、场景和风格 | 拆分角色或优先最重要的角色 |
| **FLF2V** | 从首帧过渡到尾帧 | 尾帧仅作氛围参考而非目标 | 声明 '图2' 为最终视觉目标，生成连续过渡 |
| **V2V Edit** | 保留源片段，修改一个图层 | 重写整个场景丢失连续性 | 写 '编辑视频1，仅修改[光影/背景/VFX]' |
| **V2V Extend** | 从当前末帧延续 | 开始新场景无连续性锚点 | 用末帧作为首帧锚点，每次修改一个变量 |

## Seedance I2V 核心法则

**只描述图像无法呈现的内容。** 静态图像已包含主体身份、产品形态、服装、调色板、构图、背景。重新描述这些静态细节通常导致漂移。仅为图像添加：运动、摄影机、时间推进、光线变化、声音、保留约束。

## I2V 分镜规划

遵循 Seedance 操作模式：

1. **先摄取：** 厘清目标、时长、画幅比例、参考素材、交付物。
2. **模式门：** 假定服务于 I2V 规划，除非用户说 T2V、V2V、R2V、edit 或 extend。
3. **参考素材映射：** 若有素材存在，为每个素材分配角色（身份、首帧、环境、运动、摄影机、时间、风格）。说明哪些不应传递。
4. **长视频逻辑：** 超过 15 秒的视频，生成分镜计划并注明最终 Seedance 生成应拆分为多个 Part。
5. **分镜板输入：** 若用户提供多面板板图，≤4 面板用于一条带时间戳的 Seedance 提示词，≥5 面板用于每条独立镜头提示词加剪辑节奏。

## I2V 运动备注规范

为每帧包含一条运动备注：

- **摄影机运动：** static hold, slow push-in, lateral track, gentle tilt, handheld drift。
- **主体运动：** turn, step, reach, glance, pause, reveal, react。
- **过渡逻辑：** match action, eye-line match, push through foreground, cut on gesture。
- **避免：** 一帧包含多个时间分离的动作。

## 常见 I2V 失败修复

| 失败 | 修复 |
|------|------|
| 身份漂移 | 减少新视觉描述，加强保留约束 |
| 摄影机跳变 | 使用一个摄影机运动，注明起止帧 |
| 产品变形 | 声明 preserved, static identity, no shape change |
| 画面静止 | 添加一个物理动作和一条时间提示 |
| 背景变化 | 保留环境布局，仅动画化光线/天气/氛围 |
| 手部变形 | 简化手部动作或将手部排除在主动作之外 |
