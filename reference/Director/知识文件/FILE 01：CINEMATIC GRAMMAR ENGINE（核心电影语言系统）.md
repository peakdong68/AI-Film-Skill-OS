# 🎬 FILE 01：CINEMATIC GRAMMAR ENGINE（核心电影语言系统）

---

# 🧠 1. 系统定位（System Identity）

Cinematic Grammar 是 Seedance 2.0 的“视觉语言内核”。

它的作用是：

> 将自然语言（故事/脚本）转换为可执行的电影摄影机语言（Camera Executable Language）

---

## 🎯 核心目标

系统必须实现以下转换能力：

```text
Story (文字)
→ Emotion (情绪)
→ Shot Language (镜头语言)
→ Camera Motion (摄影机运动)
→ Lighting Logic (光影结构)
→ AI Video Prompt (可执行视频指令)
```

---

# 🎥 2. 镜头语言基础结构（Shot Language Kernel）

每一个可执行镜头必须由以下 4 层构成：

---

## 2.1 镜头类型（Shot Scale System）

系统必须自动选择：

* **Wide Shot**

  * 用途：空间 / 孤独 / 环境关系
  * 功能：建立世界结构

* **Medium Shot**

  * 用途：行为 / 互动 / 关系
  * 功能：叙事主干

* **Close-Up**

  * 用途：情绪 / 心理变化
  * 功能：情感核心

* **Extreme Close-Up**

  * 用途：细节 / 决定性瞬间
  * 功能：情绪爆点 / 视觉锚点

---

## 2.2 摄影机角度系统（Camera Angle Logic）

必须从以下集合选择：

* Eye-level → 现实中立
* Low angle → 权力 / 威胁 / 重要性
* High angle → 脆弱 / 被压制
* Dutch angle → 心理不稳定 / 紧张
* Top-down → 控制 / 孤立
* Over-the-shoulder → 关系冲突 / 对话结构
* POV → 沉浸式体验

---

## 2.3 摄影机运动系统（Camera Motion Engine）

必须明确“运动意图”：

* Static frame → 观察 / 压迫 / 停顿
* Slow dolly-in → 情绪加深 / 压迫增强
* Dolly-out → 情绪抽离 / 孤独感
* Tracking shot → 跟随 / 行动 / 追逐
* Orbit shot → 情绪包围 / 关系复杂化
* Handheld → 现实感 / 不稳定 / 紧张
* Crane / Drone → 空间扩展 / 史诗感

---

## 2.4 镜头语言本质规则

每一个镜头必须回答：

1. 我在看什么（Subject）
2. 为什么这样看（Intent）
3. 摄影机如何移动（Motion）

---

# 🎭 3. 表演翻译系统（Performance Translation Engine）

## ❌ 禁止抽象表达：

* 她很伤心
* 他很紧张
* 她很开心

---

## ✅ 必须转换为可视动作：

系统必须输出：

* 身体动作
* 微表情
* 呼吸变化
* 眼神方向

---

### 示例转换：

❌ 输入：
“她很悲伤”

✔ 输出：

* 她低头
* 肩膀轻微下沉
* 呼吸变慢
* 避免眼神接触
* 手指轻微颤动

---

# 🌊 4. 情绪 → 镜头映射系统（Emotion Mapping Engine）

系统自动映射：

| 情绪 | 镜头语言                                |
| -- | ----------------------------------- |
| 爱情 | slow dolly-in + warm light          |
| 恐惧 | handheld + close-up + flicker light |
| 孤独 | wide shot + negative space          |
| 紧张 | close-up + Dutch angle              |
| 神秘 | silhouette + static + backlight     |
| 震惊 | extreme close-up + sudden stillness |

---

# 🌫 5. 光影语法系统（Lighting Grammar Engine）

光影必须服务叙事：

---

## 5.1 光影类型

* Warm light → 情感 / 安全 / 亲密
* Cold blue → 孤独 / 理性 / 夜晚
* High contrast → 冲突 / 压迫
* Soft diffused → 梦境 / 回忆
* Flicker light → 不稳定 / 恐惧

---

## 5.2 光影规则

* 光 = 情绪驱动
* 阴影 = 未知 / 压力
* 反光 = 现实扭曲
* 背光 = 神秘 / 情绪隔离

---

# 🧠 6. 镜头叙事逻辑（Visual Story Logic Engine）

每个镜头必须包含：

### ✔ 三核心问题

1. Subject（主体）
2. Emotion（情绪）
3. Intent（导演意图）

---

### ✔ 叙事原则：

* 每个镜头只能表达一个情绪节点
* 每个镜头必须推进时间或情绪
* 禁止“无意义镜头”

---

# 🎬 7. 摄影机叙事规则（Camera Narrative System）

摄影机不是记录工具，而是“情绪主体”。

---

## 核心规则：

* 推进（push-in） = 情绪加深
* 拉远（pull-out） = 情绪疏离
* 静止（static） = 情绪压迫
* 旋转（orbit） = 情绪复杂化

---

# 🔁 8. 连续性系统（Continuity Lock Engine）

必须强制锁定：

---

## 8.1 角色一致性

* face identity locked
* wardrobe fixed
* age stable

---

## 8.2 空间一致性

* 同一地点不能随机重置
* 光源方向必须一致
* 时间流必须连续

---

## 8.3 风格一致性

* 不允许风格跳变
* 不允许现实→卡通切换
* 不允许材质突变

---

# 🎥 9. Cinematic → Seedance 转换规则

这是核心压缩系统：

```text
Story
→ Shot Breakdown
→ Camera Language
→ Motion Mapping
→ Lighting Mapping
→ Prompt Compression
→ Seedance Output
```

---

# 🧾 10. 输出标准（Output Contract）

最终输出必须满足：

✔ 可直接用于 image-to-video
✔ 每个镜头独立可执行
✔ 包含 camera + motion + lighting + emotion
✔ 无解释性语言
✔ 全部为执行型描述
